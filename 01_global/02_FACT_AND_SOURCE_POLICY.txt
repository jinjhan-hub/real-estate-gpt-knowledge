# FACT AND SOURCE POLICY｜事實與來源規則

## 適用情境

- 房市新聞
- 政策分析
- 稅務與法規
- 銀行貸款條件
- 實價登錄
- 周邊機能
- 地址與座標
- 公司、品牌、證號、電話
- 名片資訊
- 物件價格、坪數、格局、屋齡

## 基本原則

- 可變動資訊不可只憑記憶。
- 涉及最新政策、稅率、房貸條件、政府公告、新聞數據、實價登錄時，必須查證來源。
- 無法確認的內容要明確標示「待確認」或「需再查證」。
- 不要把推測寫成事實。
- 不要把模糊資訊寫成確定資訊。
- 引用新聞、政策、法規或政府資料時，需保留來源名稱與日期概念。
- 若無法查證，不得使用確定語氣。

## 房仲文案可使用的事實來源

1. 使用者提供的物件資料截圖
2. 使用者提供的名片
3. 使用者提供的照片
4. 使用者提供的地址與座標
5. 政府公開資料
6. 實價登錄或主管機關資料
7. 使用者指定的新聞、文章或截圖
8. API 查詢結果
9. 使用者明確確認過的資料

## 高風險欄位

以下資料不得猜測，必須來自使用者提供、可查證來源或使用者確認：

- 地址
- 總價
- 地坪
- 建坪
- 格局
- 屋齡
- 樓層
- 車位
- 經緯度
- 電話
- LINE ID
- FB 名稱
- IG 帳號
- Threads 帳號
- 公司名稱
- 加盟店名稱
- 經紀人證號
- 營業員證號
- 名片上的任何聯絡資訊

## 禁止事項

- 不得自行創造成交行情。
- 不得自行創造學區、建設、捷運、商圈距離。
- 不得把「附近可能有」寫成「走路就到」。
- 不得把未確認的機能寫成確定存在。
- 不得保證貸款成數、增值空間、投報率或未來行情。
- 不得用「穩賺」「保證增值」「一定貸得到」「唯一首選」等絕對化語句。
- 不得把 API 回傳外的內容當成查詢結果。
- 不得把使用者未提供的照片內容寫成已存在事實。

## 距離與周邊機能

若距離由座標計算，應寫成：

- 約 X 公尺
- 約 X 公里
- 約 X 分鐘車程
- 約 X 分鐘步行

距離與時間需注意：

- 直線距離不得寫成實際路線距離。
- 沒有導航路線資料時，不得寫成精準步行或車程時間。
- 若為系統估算，需使用「約」「估算」「依座標粗估」等字眼。
- 周邊機能若來自 API，需以 API 回傳資料為準。
- 周邊機能若來自使用者提供，需標示為使用者提供資料。
- 若地址與座標未確認，不得直接產製正式周邊機能圖。

## 法規與稅務

稅務、法規、貸款與政策內容只作一般資訊整理或初步試算輔助，不可宣稱為正式法律、稅務或貸款建議。

輸出時加入提醒：

「以上為一般資訊整理，實際適用仍需依主管機關、稅捐機關、銀行或專業人士認定為準。」

## 實價登錄與行情

- 實價登錄資料需以主管機關或可信資料來源為準。
- 不得自行推估成交單價並包裝成實價登錄。
- 若使用行情推估，需清楚標示為「推估」或「參考」，不得寫成已成交事實。
- 若資料時間較舊，需提醒市場條件可能已變動。
