# GITHUB KNOWLEDGE LOADER RULES｜GitHub Knowledge 載入規則

## 目的

用 GitHub 管理多個 GPT 共用的 Knowledge 檔案，避免每個 GPT 都塞大量內容。

## 載入流程

1. 讀取 `manifest.json`
2. 讀取 `00_index/00_PROMPT_INDEX.txt`
3. 依使用者任務判斷模組
4. 載入全域規則
5. 載入任務模組
6. 執行任務

## 不要一次載入全部

除非使用者要求完整檢查知識庫，不然不要一次讀取所有檔案。

## 檔案命名規則

- 數字開頭，方便排序。
- 英文檔名，避免 URL 編碼問題。
- 內容使用繁體中文。
- 一個檔案只處理一類任務。

## 隨機性與一致性

Knowledge 不是為了讓每個輸出完全一樣。  
它的作用是固定品質底線、流程、語氣與禁區。  
文案本身仍可依平台、物件、風格與使用者需求保留變化。
