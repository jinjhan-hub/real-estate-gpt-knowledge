# NEARBY API CONTRACT｜周邊機能 API 契約

## Endpoint

`POST /api/nearby-facilities`

本 API 用於依地址與經緯度查詢周邊生活機能。  
地址主要用於顯示與確認，實際查詢以經緯度為主。

---

## API 定位

- API 只負責查詢與回傳周邊機能資料。
- API 不負責圖片生成、圖片合成、PNG 輸出或 PDF 輸出。
- GPTs 取得 API 回傳後，負責整理成清單、資訊卡文案或銷售圖卡可用短句。

---

## Request Body

```json
{
  "address": "彰化縣員林市中山南路8號",
  "lat": 23.9560419,
  "lng": 120.5709458,
  "radius": 500,
  "categories": ["school", "transport", "shopping", "park", "medical"]
}
```

## 欄位說明

- `address`：使用者提供地址。
- `lat`：緯度。
- `lng`：經度。
- `radius`：查詢半徑，單位公尺，預設 500。
- `categories`：查詢類別，預設一次查詢五種固定類別：school、transport、shopping、park、medical。

不得使用 `latitude` / `longitude` 作為 request 欄位，除非 Vercel API 與 OpenAPI 已同步改版。

---

## categories 固定類別

固定使用英文類別，避免 API 接收中文造成錯誤。

目前支援：

- `school`：教育 / 學校
- `transport`：交通
- `shopping`：採買
- `park`：公園 / 休閒
- `medical`：醫療

目前不列為 API 固定支援類別：

- `food`
- `finance`
- `convenience_store`
- 其他未定義分類

若使用者輸入中文類別，GPT 應先轉換成對應英文類別，再呼叫 API。

中文轉換建議：

- 學校 / 教育 → `school`
- 交通 / 車站 / 公車 / 交流道 → `transport`
- 採買 / 市場 / 超市 / 便利商店 → `shopping`
- 公園 / 休閒 / 綠地 → `park`
- 醫療 / 診所 / 醫院 / 藥局 → `medical`

預設一次查詢五種固定分類：school、transport、shopping、park、medical。若使用者明確只要求部分分類，可只帶入該部分固定類別；若使用者要求固定支援類別以外的分類，需先說明 API 尚未支援。

---

## Request 驗證規則

API 應檢查：

- `address` 是否存在。
- `lat` 是否為數字。
- `lng` 是否為數字。
- `lat` 是否落在台灣合理緯度範圍，約 21 至 26。
- `lng` 是否落在台灣合理經度範圍，約 119 至 123。
- `radius` 是否為正數。
- `categories` 是否為支援類別。
- `categories` 是否包含未支援類別。

若資料不完整，API 應回傳錯誤訊息，不應假裝查詢成功。

```json
{
  "query_status": "success",
  "address": "彰化縣員林市中山南路8號",
  "center": {
    "lat": 23.9560419,
    "lng": 120.5709458
  },
  "radius": 500,
  "distance_type": "straight_line_estimate",
  "facilities": [
    {
      "category": "shopping",
      "category_label": "採買",
      "name": "便利商店名稱",
      "distance_m": 350,
      "source": "overpass"
    }
  ],
  "summary": "周邊採買與交通設施可作為生活圈參考，距離為依座標粗估。",
  "notes": [
    "距離為依座標估算，實際路線與時間仍需依地圖導航及現場狀況為準。"
```

若部分類別查詢不到結果：

```json
{
  "query_status": "partial",
  "address": "彰化縣員林市中山南路8號",
  "center": {
    "lat": 23.9560419,
    "lng": 120.5709458
  },
  "radius": 500,
  "distance_type": "straight_line_estimate",
  "facilities": [
    {
      "category": "shopping",
      "category_label": "採買",
      "name": "便利商店名稱",
      "distance_m": 350,
      "source": "overpass"
    }
  ],
  "missing_categories": ["medical"],
  "notes": [
    "部分分類查詢結果有限，未列出的設施不代表實際不存在。"
  ]
}
```

---

## Error Response 建議

### 缺少座標

```json
{
  "query_status": "error",
  "error_code": "MISSING_COORDINATES",
  "message": "缺少經緯度，無法穩定查詢周邊機能。"
}
```

### 座標格式錯誤

```json
{
  "query_status": "error",
  "error_code": "INVALID_COORDINATES",
  "message": "座標格式可能有誤，請確認是否為緯度、經度。"
}
```

### 分類超出固定支援範圍

```json
{
  "query_status": "error",
  "error_code": "UNSUPPORTED_CATEGORY_SCOPE",
  "message": "目前固定支援 categories 為 school、transport、shopping、park、medical；請移除未支援分類後再查詢。"
}
```

### 不支援的分類

```json
{
  "query_status": "error",
  "error_code": "UNSUPPORTED_CATEGORY",
  "message": "包含 API 尚未支援的 categories，請改用 school、transport、shopping、park 或 medical。"
}
```

### 外部服務失敗

```json
{
  "query_status": "error",
  "error_code": "EXTERNAL_SERVICE_ERROR",
  "message": "外部查詢服務暫時無回應，請稍後再試。"
}
```

### 查無資料

```json
{
  "query_status": "empty",
  "message": "目前查詢結果有限，未取得可確認的周邊設施。"
}
```

---

## GPT 輸出規則

API 回傳後，GPT 不直接丟 JSON 給一般使用者。  
應整理成：

- 分類清單
- 約略距離
- 生活圈觀察
- 圖卡可用短句
- 注意事項

## GPT 不得做的事

- 不得把 API 沒回傳的設施補進結果。
- 不得把直線距離寫成實際路線距離。
- 不得把查詢結果寫成 Google Maps 導航時間。
- 不得把「附近有學校」寫成「明星學區」。
- 不得把「附近有商家」寫成「成熟商圈」，除非另有資料依據。
- 不得在 `query_status` 為 `error` 或 `empty` 時，輸出完整周邊機能清單。
- 不得使用 `food`、`finance` 等未支援 categories 呼叫 API。

### 採買
