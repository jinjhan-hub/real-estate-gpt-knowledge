# IMAGE GENERATION WORKFLOW｜圖片生成工作流程

用途：控制銷售圖卡、FB圖文、IG圖文與其他圖片生成流程。

## 三段式流程

1. 資料確認階段
   - 辨識物件資料、照片、名片與聯絡資訊。
   - 列出已確認資料與待確認資料。
   - 不得在此階段生成圖片。

2. 產圖設定階段
   - 確認平台、尺寸、風格與圖卡模式。
   - 判斷是否可使用人物、QR Code、名片資訊。
   - 必要時呼叫 API 讀取對應文字規則。
   - 整理短版產圖摘要。

3. 圖片生成階段
   - 只生成一張圖片。
   - 圖片生成時不得同時產出完整 FB / IG / Threads 文案。
   - 圖片完成後，再接續或詢問是否生成社群文案。

## 規則

- 不得自行生成未提供的房屋照片。
- 不得改變房屋結構。
- 不得生成錯誤電話、證號或公司資訊。
- 若小字可能錯誤，優先生成不含小字版本或提醒後製。
- QR Code 只能使用使用者提供且清楚可辨識的素材，不得自行重畫。
- 人物只能使用使用者提供且可辨識的人物照片或名片人像，不得自行生成。

## 圖卡應優先

- 主視覺
- 重點資訊
- 可讀性
- 手機瀏覽體驗

## 避免

- 過度塞滿資訊
- 圖文混合過多
- FB / IG / Threads 文案直接放進圖片
- 同一輪同時產圖與產完整文案
