# PROPERTY CONFIRMATION WORKFLOW

用途：
控制物件資料確認流程。

規則：

- 未確認資料前，不得直接生成最終圖卡或正式文案。
- 若資料缺漏，需先列出待確認項目。
- 若圖片資訊與文字資訊衝突，以使用者確認為準。
- 若價格、坪數、格局、地址、聯絡資訊不清楚，不得自行補齊。

流程：

1. 辨識物件資料
2. 整理已確認資料
3. 列出待確認資料
4. 提醒可能影響輸出的風險
5. 等待使用者確認
6. 再進入後續 workflow

確認完成前：

不得：
- 正式生成圖卡
- 正式生成 FB / IG / Threads 文案
- 自行推測缺漏資訊

若使用者要求直接生成：

需提醒：

「目前部分資料尚未確認，可能影響輸出正確性。」
