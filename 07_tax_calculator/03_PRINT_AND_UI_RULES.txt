# PRINT AND UI RULES｜列印與介面規格

本檔定義稅務試算器的手機介面、列印版、PDF 輸出與 GitHub Pages 部署規則。

適用頁面：

- `index.html`：房地合一稅試算器
- `land-tax-calculator.html`：土地增值稅試算器
- 入口頁面，若使用者另行建立

---

## Mobile-first

介面需以手機使用為優先。

### 欄位設計

- 手機上欄位要大。
- 輸入框高度需足夠，避免難以點選。
- 字體不可太小。
- 數字欄位應使用適合手機輸入的格式。
- 日期欄位需有清楚格式提示。
- 金額欄位需有單位提示，例如「元」。
- 坪數、平方公尺、每平方公尺等單位需標示清楚。

### 按鈕設計

- 按鈕距離要足夠，避免誤觸。
- 「帶入範例」「清除」「開始試算」三個按鈕不可太靠近。
- 主要按鈕需明顯。
- 危險操作，例如清除資料，需避免使用者誤點。
- 手機上按鈕需適合拇指操作。

### 閱讀性

- 表單區塊需分段。
- 結果區要清楚。
- 錯誤提醒要靠近相關欄位。
- 不要讓使用者滑太久才看到重要結果。
- 手機版不使用過寬表格。

---

## 結果區

- 結果要在同頁清楚呈現。
- 結果區需出現在使用者點擊「開始試算」後。
- 結果區需包含輸入摘要、計算結果、注意事項與免責提醒。
- 列印按鈕放在試算結果最下方。
- 結果可直接列印或另存 PDF。
- 若欄位缺漏或格式錯誤，不得產出正式試算結果。
- 若試算涉及未確認條件，結果區需標示「待確認」或「僅供初步參考」。

---

## A4 列印

### 基本規則

- 使用白底。
- 不要深色背景。
- 字體大小足夠。
- 表格不可超出頁面。
- 列印版需適合 A4。
- 頁面邊界需保留足夠空間。
- 不要讓重要結果被分頁切斷。
- 列印版不顯示不必要的操作按鈕。

### 列印內容

列印版需包含：

- 頁面名稱
- 品牌文字
- 輸入資料摘要
- 試算結果
- 成本或費用明細，若有
- 注意事項
- 免責提醒
- 製作來源或浮水印

### 浮水印規則

- 浮水印需在最上層。
- 浮水印不得遮住重要數字。
- 浮水印不得影響結果可讀性。
- 浮水印文字需依使用者最新提供品牌資料為準。
- 若浮水印與表格重疊，應降低透明度或調整位置，但仍需可見。

---

## PDF 輸出

- 使用瀏覽器列印功能另存 PDF。
- PDF 版面需與 A4 列印一致。
- 不得出現深色背景。
- 不得遺漏免責提醒。
- 不得讓結果表格被截斷。
- 若使用者要求下載 PDF，優先確保列印 CSS 正確。

---

## CSS 規則

需分開設計：

### 螢幕版 CSS

- 適合手機瀏覽。
- 表單清楚。
- 按鈕大、間距足夠。
- 結果區易讀。

### 列印版 CSS

使用 `@media print` 控制：

- 隱藏操作按鈕。
- 使用白底。
- 調整字體大小。
- 避免表格超出 A4。
- 控制分頁。
- 顯示浮水印。
- 保留免責提醒。

---

## GitHub Pages

- 靜態 HTML / CSS / JS 優先。
- 優先使用單檔 HTML，方便部署與維護。
- 不依賴後端服務，除非使用者明確要求 API 串接。
- 檔名避免中文與空格。
- 檔名避免特殊符號。
- 修改檔名時，需同步更新入口頁連結。

入口頁可連結：

- `index.html`
- `land-tax-calculator.html`

若使用者已部署 GitHub Pages，不得任意更改檔名或路徑，避免原本連結失效。

---

## 修改既有程式時的限制

- 不得任意刪除原本功能。
- 不得移除列印功能。
- 不得移除 PDF 需求。
- 不得移除浮水印。
- 不得移除免責提醒。
- 不得破壞手機版排版。
- 不得破壞 GitHub Pages 可部署性。
- 修改 UI 時需保留原本試算邏輯，除非使用者明確要求調整公式。
- 修改公式時需說明修改原因與影響範圍。

---

## 免責提醒位置

免責提醒需出現在：

- 網頁結果區下方
- A4 列印版下方
- PDF 輸出版本下方

建議文字：

「本工具僅供初步試算與教學參考，實際稅額、適用稅率、免稅條件與申報結果，仍需依主管機關、稅捐機關或專業人士認定為準。」
