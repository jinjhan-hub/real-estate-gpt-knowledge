# 05｜Final Image Prompt Rule
# ruleKey: 05_final_image_prompt
# requiredStage: FINAL_IMAGE_PROMPT
# nextAction: saveFinalImagePrompt
# mustCallNext: saveFinalImagePrompt

本規則用於「最終圖片提示詞整理階段」。

本階段目標：
1. 將已確認的物件資料、風格選擇、合規揭露資料整理成 final_image_prompt。
2. 明確限制圖片生成不得產生 QR Code。
3. 明確限制圖片生成不得產生未授權證號、地址、電話或揭露文字。
4. 確保圖片生成提示詞只使用已確認來源的資料。
5. 儲存 final_image_prompt 後，才可進入圖片政策與 failsafe 階段。

---

## 一、流程硬鎖

進入本階段時，必須遵守：

1. 不得呼叫 saveImagePolicies，除非 final_image_prompt 已完成。
2. 不得呼叫 generateImage。
3. 不得呼叫 completeGeneration。
4. 不得跳過 final_image_prompt 儲存。
5. 不得使用其他 session 的資料。
6. 不得使用測試案例殘留資料。
7. 不得自行補齊任何證號、電話、地址或揭露文字。
8. 不得要求 AI 生成 QR Code。

本階段完成條件：

1. final_image_prompt 已依已確認資料完成。
2. final_image_prompt 內已包含 QR Code 放置區規則。
3. final_image_prompt 內不得要求生成 QR Code。
4. final_image_prompt 內不得包含來源不明的證號、電話、地址或揭露文字。
5. final_image_prompt 已呼叫 saveFinalImagePrompt 保存。

---

## 二、final_image_prompt 可使用資料來源

final_image_prompt 只能使用以下已確認資料：

1. 使用者明確提供並確認的物件資料。
2. 使用者明確提供並確認的銷售重點。
3. 名片上清楚可辨識且已確認的文字。
4. 使用者明確輸入的營業員與公司聯絡資料。
5. API disclosure 回傳的公司 / 店家 / 經紀人揭露資料。
6. 前一階段合規檢查通過的資料。
7. 使用者已選定的圖片風格。

禁止使用：

1. GPT 自行生成的營業員姓名。
2. GPT 自行生成的營業員電話。
3. GPT 自行生成的營業員證號。
4. GPT 自行生成的公司地址。
5. GPT 自行生成的公司電話。
6. GPT 自行生成的經紀業字號。
7. GPT 自行生成的經紀人字號。
8. 其他案例殘留資料。
9. 其他 session 資料。
10. 測試案例資料。
11. 看似正式但未經來源確認的證號、電話或地址。

---

## 三、QR Code 放置區規則

無論使用者是否提供 QR Code，圖片生成階段一律不得生成、重繪、仿製或嵌入 QR Code 圖樣。

若銷售圖卡需要 QR Code：

1. 圖片中只能預留 QR Code 放置區。
2. QR Code 放置區應為乾淨白底或淺色方框。
3. 放置區內可標示「QR Code 放置區」或「掃碼加 LINE」等文字。
4. 不得產生任何類似 QR Code 的黑白方塊圖樣。
5. 不得產生假 QR Code。
6. 不得描述 QR Code 外觀讓 AI 自行生成。
7. 真正 QR Code 應由後製流程或人工另行貼上原始 QR Code 圖檔。
8. final_image_prompt 應明確寫出：「請預留 QR Code 放置區，不要生成 QR Code 圖樣」。

final_image_prompt 中必須包含以下明確句：

```text
請預留 QR Code 放置區，不要生成 QR Code 圖樣。
```

final_image_prompt 中不得包含：

```text
生成 QR Code
繪製 QR Code
放入 QR Code 圖樣
產生可掃描 QR Code
仿製 QR Code
重繪 QR Code
```

## 四、營業員與公司聯絡資訊輸出規則

final_image_prompt 中的營業員與公司聯絡資訊，只能使用以下來源：

1. 名片上清楚可辨識的文字
2. 使用者明確輸入的資料
3. API disclosure 回傳的資料

可輸出欄位：

1. 營業員姓名
2. 營業員電話
3. 營業員證號
4. 公司地址
5. 公司電話
6. 店名
7. 公司名稱
8. 經紀人姓名
9. 經紀人字號
10. 經紀業相關字號

禁止事項：

1. 不得自行生成營業員姓名。
2. 不得自行生成營業員電話。
3. 不得自行生成營業員證號。
4. 不得自行生成公司地址。
5. 不得自行生成公司電話。
6. 不得自行生成經紀業字號。
7. 不得自行生成經紀人字號。
8. 不得使用其他案例殘留資料。
9. 不得使用看似正式但未經來源確認的證號或電話。
10. 不得補齊名片上模糊不清的數字。
11. 不得將經紀人字號誤當成營業員證號。
12. 不得將營業員證號誤當成經紀業字號。
13. 不得將公司電話誤當成營業員手機，除非來源明確。
14. 不得將營業員手機誤當成公司電話，除非來源明確。

若營業員證號未由名片、使用者輸入或 API disclosure 提供：

1. final_image_prompt 不得輸出營業員證號。
2. 必須標記缺少營業員證號供合規檢查處理。
3. 不得自行補上一組看似正式的字號。

---

## 五、公司 / 店家 / 經紀人揭露資料輸出規則

公司 / 店家 / 經紀人揭露資料優先使用 API disclosure。

包含：

1. 公司名稱
2. 店家名稱
3. 加盟店名稱
4. 經紀人姓名
5. 經紀人字號
6. 經紀業相關字號
7. API disclosure 內明確提供的揭露資料

禁止：

1. 不得由 GPT 根據店名推測公司名稱。
2. 不得由 GPT 根據品牌推測加盟店名稱。
3. 不得由 GPT 根據縣市推測字號。
4. 不得使用其他店家資料。
5. 不得使用測試案例資料。
6. 不得使用記憶中的資料補齊。
7. 不得輸出未經來源確認的經紀業字號。

若 API disclosure 未提供某項揭露資料：

1. 不得自行生成。
2. 不得自行補齊。
3. 必須標記為缺少或待補充。

---

## 六、物件文字輸出規則

final_image_prompt 可包含以下物件文字：

1. 主標
2. 副標
3. 價格
4. 區域
5. 土地坪數
6. 建物坪數
7. 格局
8. 車位
9. 屋齡
10. 物件亮點
11. 聯絡資訊
12. 揭露資訊

物件文字必須遵守：

1. 使用已確認資料。
2. 不得自行補上未確認特色。
3. 不得誇大。
4. 不得使用保證性用語。
5. 不得使用絕對性用語。
6. 不得承諾增值。
7. 不得承諾貸款。
8. 不得承諾投報。
9. 不得生成不存在的生活機能。
10. 不得生成不存在的交通設施。

---

## 七、圖片生成版面規則

final_image_prompt 可描述版面方向，但不得違反素材限制。

建議包含：

1. FB 4:5 銷售圖卡。
2. 台灣房仲廣告版面。
3. 主視覺以使用者提供的物件照片為主。
4. 文字層級清楚。
5. 資訊區乾淨易讀。
6. 預留 QR Code 放置區。
7. 聯絡資訊放置於底部或側邊。
8. 揭露資訊清楚但不喧賓奪主。
9. 不生成假人物。
10. 不生成假 QR Code。

若使用者沒有提供人物照：

```text
不得生成真人業務人物。
```

若使用者有提供人物照：

```text
需遵守人物照使用規則，不得改變人物身份，不得生成與原人物不一致的真人。
```

---

## 八、final_image_prompt 必須包含的安全限制

final_image_prompt 必須包含以下限制文字：

```text
圖片生成限制：
1. 請使用使用者提供的物件照片作為主視覺，不要替換成其他房屋。
2. 不要生成假人物。
3. 若未提供人物照，不要生成真人業務形象。
4. 請預留 QR Code 放置區，不要生成 QR Code 圖樣。
5. 不要生成、重繪、仿製或嵌入 QR Code。
6. 不要生成任何未提供的營業員證號、經紀人字號、經紀業字號、公司地址、公司電話或營業員電話。
7. 所有證號、地址、電話與揭露資訊，必須完全依照已確認資料輸出。
8. 不要使用測試案例、範例資料或其他店家資料。
```

---

## 九、final_image_prompt 建議格式

final_image_prompt 建議使用以下結構：

```text
請生成一張 FB 4:5 台灣房仲銷售圖卡。

【風格】
{selectedStyle 的風格描述}

【主視覺】
使用使用者提供的物件照片作為主視覺。
不要替換成其他房屋。
不要生成假物件照片。

【主要文字】
主標：{物件主標}
副標：{副標}
資訊列：{價格｜區域｜坪數｜格局｜車位等}

【物件亮點】
1. {亮點一}
2. {亮點二}
3. {亮點三}

【聯絡資訊】
營業員：{營業員姓名}
電話：{營業員電話}
營業員證號：{營業員證號，若未提供則不得輸出}
公司地址：{公司地址，若未提供則不得輸出}
公司電話：{公司電話，若未提供則不得輸出}

【揭露資訊】
公司名稱：{API disclosure 公司名稱}
經紀人：{API disclosure 經紀人姓名}
經紀人字號：{API disclosure 經紀人字號}
其他揭露資訊：{API disclosure 明確提供資料}

【QR Code 區域】
請在版面中預留 QR Code 放置區。
請預留 QR Code 放置區，不要生成 QR Code 圖樣。
QR Code 放置區應為乾淨白底或淺色方框，可標示「QR Code 放置區」或「掃碼加 LINE」。
不得生成、重繪、仿製或嵌入 QR Code。
不得產生任何類 QR Code 黑白方塊圖樣。

【圖片生成限制】
1. 不要生成假人物。
2. 若未提供人物照，不要生成真人業務形象。
3. 不要生成 QR Code。
4. 不要生成任何未提供的證號、地址、電話。
5. 不要使用測試案例或其他店家資料。
6. 所有文字必須依照已確認資料。
```

---

## 十、final_image_prompt 不得包含的內容

不得包含：

1. 生成 QR Code。
2. 可掃描 QR Code。
3. 仿製 QR Code。
4. 重繪 QR Code。
5. AI 自行設計 QR Code。
6. 未提供的營業員證號。
7. 未提供的經紀業字號。
8. 未提供的經紀人字號。
9. 未提供的公司地址。
10. 未提供的公司電話。
11. 測試業務。
12. 測試電話。
13. 測試字號。
14. 其他店家資料。
15. 假人物描述。
16. 假物件照片描述。

---

## 十一、本階段完成判定

只有在以下條件全部成立時，才可呼叫 saveFinalImagePrompt：

1. final_image_prompt 已完成。
2. final_image_prompt 只使用已確認資料。
3. final_image_prompt 已包含 QR Code 放置區規則。
4. final_image_prompt 未要求生成 QR Code。
5. final_image_prompt 未包含來源不明的證號、電話、地址或揭露文字。
6. final_image_prompt 未使用測試案例資料。
7. final_image_prompt 未使用其他店家資料。
8. final_image_prompt 已明確禁止生成假人物。
9. final_image_prompt 已明確禁止生成假 QR Code。

完成後才可呼叫：

```text
saveFinalImagePrompt
```
