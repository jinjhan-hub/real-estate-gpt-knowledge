# 05_final_image_prompt｜最終圖片提示詞建立規則

## 本階段責任

本階段只負責根據已確認物件資料、合規結果與已選風格建立 `finalImagePrompt`。

不得補未提供資訊，不得生成圖片，不得呼叫 completeGeneration。

## 可使用資料來源

只能使用以下資料：
1. 已經使用者確認並儲存的物件資料。
2. 已通過合規檢查的文字內容。
3. 已選定的 `selectedStyle`。
4. 已記錄的素材狀態文字，例如是否有房屋照片、人物照、QR Code。
5. 使用者明確提供並已確認的店家、經紀業、營業員、聯絡資料。

不得使用：
- 未確認的聊天內容
- 模型自行推測的房產資訊
- 從圖片推測出的房產資訊
- 外部資料
- 未提供的店家資料
- 未提供的聯絡資訊

## finalImagePrompt 必須遵守的限制

### 一、不得新增未提供資訊

禁止加入：
- 電話
- LINE ID
- Email
- 完整地址
- 地坪
- 建坪
- 屋齡
- 樓層
- 學區
- 商圈
- 交通時間
- 交流道距離
- 市場、公園、醫院等機能資訊

除非該資訊已在已確認物件資料中存在。

### 二、不得生成不存在的房屋實景

若使用者有提供房屋照片：
- 可以要求以使用者提供的房屋照片作為主要視覺素材。
- 不得重畫成不同房屋。
- 不得生成不存在的外觀、室內、庭院、車庫或街景。

若使用者未提供房屋照片：
- 不得生成具體房屋實景。
- 可使用抽象質感背景、色塊、幾何排版、生活感氛圍背景。
- 不得假裝有真實物件照片。

### 三、人物使用限制

若使用者有提供人物照：
- 可保留人物作為版面素材。
- 不得改變人物五官、臉型、身形比例、年齡感與主要特徵。
- 不得生成不相干人物。

若使用者未提供人物照：
- 不得生成虛構人物。
- 不得加入房仲肖像、買方、家庭、路人或模特兒。

### 四、QR Code 使用限制

若使用者有提供 QR Code：
- 必須預留乾淨可放置區。
- 不得重畫 QR Code。
- 不得偽造 QR Code。
- 不得改變 QR Code 結構。
- 不得讓 QR Code 變形、模糊、遮擋或被文字覆蓋。

若使用者未提供 QR Code：
- 不得生成假 QR Code。
- 不得加入仿 QR Code 圖形。
- 不得加入掃碼框、掃一掃、Scan me 等引導掃碼元素。

## finalImagePrompt 建議結構

請產出一段可直接給圖片生成模型使用的提示詞，結構如下：

```text
請生成一張 FB 4:5 房仲銷售圖卡。

風格：{selectedStyle 對應的中文風格描述}

主要文字：
- 主標：{案名}
- 副標：{已確認可用的核心賣點}
- 資訊列：{價格、地點、格局、車位等已確認資訊}
- 聯絡資訊：{已確認可用的聯絡資訊；未提供則不要加入}
- 揭露資訊：{已確認可用的經紀業、經紀人、營業員字號等；未提供則不要加入}

版面要求：
- 直式 4:5
- 資訊清楚易讀
- 留白充足
- 不要過度裝飾
- 不要新增未提供資訊

素材限制：
- 若有房屋照片，僅能使用使用者提供的房屋照片，不得生成不存在的房屋實景。
- 若有 QR Code，只預留放置區，不得重畫、不偽造、不變形。
- 若有人物照，人物五官、臉型、身形比例不得變形。
- 若未提供人物照，不得生成虛構人物。

禁止事項：
- 不新增電話、LINE ID、Email、完整地址。
- 不自動補坪數、屋齡、樓層、學區、商圈、交通資訊。
- 不生成不存在的房屋實景。
- 不生成假 QR Code。
- 不生成假人物。
```

## selectedStyle 視覺轉換

### luxury_premium

```text
高質感精品感、穩重、乾淨、深色或中性色基調、留白充足、精品型房產廣告風格、文字層級清楚。
```

### clean_modern

```text
乾淨現代感、明亮、簡約、資訊清楚、適合社群閱讀、排版俐落。
```

### warm_home

```text
溫暖居家感、柔和色調、親切、生活感、適合家庭住宅與換屋族。
```

### commercial_focus

```text
商用機能感、俐落、資訊導向、條件明確、適合投資型或商用不動產。
```

## finalImagePrompt 儲存格式

完成後應呼叫：

```text
saveFinalImagePrompt
```

建議儲存欄位：

```json
{
  "sessionId": "string",
  "finalImagePrompt": "string"
}
```

## 禁止行為

本階段不得：
- 建立 QR Code policy、portrait policy、failsafe policy
- 生成圖片
- 檢查圖片結果
- 呼叫 completeGeneration
- 儲存圖片、PDF、QR Code、人物照、房屋照片等非文字素材

## 本階段完成條件

建立 finalImagePrompt 後，必須呼叫：

```text
saveFinalImagePrompt
```

## nextInstruction

請根據本階段規則，只使用已確認物件資料、合規後文字與 selectedStyle 建立 finalImagePrompt。不得補未提供資訊，不得加入電話、LINE ID、完整地址、學區、商圈、交通資訊。完成後必須呼叫 saveFinalImagePrompt。
