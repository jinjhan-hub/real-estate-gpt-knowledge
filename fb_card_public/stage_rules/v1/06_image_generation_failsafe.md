# 06_image_generation_failsafe｜圖片生成保護規則

## 本階段責任

本階段只負責建立圖片生成前的保護政策。

本階段完成後，才可進入 `IMAGE_GENERATION_READY`。

不得在本階段直接呼叫 completeGeneration。

## 必須建立的保護政策

本階段應建立以下三類政策：

```json
{
  "qrcodePolicy": "",
  "portraitPolicy": "",
  "failsafePolicy": ""
}
```

## QR Code 政策

### 若使用者有提供 QR Code

必須遵守：

1. QR Code 不得被重畫。
2. QR Code 不得被偽造。
3. QR Code 不得被模型重新生成。
4. QR Code 不得被改變結構。
5. QR Code 不得被拉伸、壓縮、旋轉、扭曲。
6. QR Code 不得被文字、人物、裝飾物、色塊遮擋。
7. QR Code 周圍必須保留乾淨留白。
8. 只能預留 QR Code 放置區或保留使用者原始 QR Code 作為素材。
9. 不得解析、儲存或改寫 QR Code 內容。

建議政策文字：

```text
若使用者有提供 QR Code，圖片生成時只能保留原始 QR Code 或預留乾淨可放置區。不得重畫、偽造、變形、模糊、遮擋或改變 QR Code 結構；不得生成新的 QR Code；不得儲存 QR Code 圖片本身。
```

### 若使用者未提供 QR Code

必須遵守：

1. 不得生成假 QR Code。
2. 不得生成仿 QR Code 圖案。
3. 不得放入看似可掃描的圖碼。

建議政策文字：

```text
若使用者未提供 QR Code，不得生成任何假 QR Code、仿 QR Code 圖案或看似可掃描的圖碼。
```

## 人物政策

### 若使用者有提供人物照

必須遵守：

1. 人物五官不得變形。
2. 臉型不得變形。
3. 身形比例不得變形。
4. 年齡感不得被明顯改變。
5. 不得生成不同人物替代原人物。
6. 不得過度美化到失真。
7. 不得改變人物主要辨識特徵。
8. 不得將人物套用成卡通、動漫、插畫或不符合原照片的風格，除非使用者明確要求。
9. 不得儲存人物照本身。

建議政策文字：

```text
若使用者有提供人物照，圖片生成時必須保留人物原本五官、臉型、身形比例、年齡感與主要辨識特徵。不得變形、不得生成替代人物、不得過度美化到失真，也不得儲存人物照本身。
```

### 若使用者未提供人物照

必須遵守：

1. 不得生成虛構房仲人物。
2. 不得生成買方、家庭、路人、模特兒。
3. 不得生成任何具體人物肖像。

建議政策文字：

```text
若使用者未提供人物照，不得生成虛構人物、房仲肖像、買方、家庭、路人或模特兒。
```

## 房屋照片政策

### 若使用者有提供房屋照片

必須遵守：

1. 僅能使用使用者提供的房屋照片作為房屋視覺素材。
2. 不得重畫成不同房屋。
3. 不得新增不存在的外觀。
4. 不得新增不存在的室內裝潢。
5. 不得新增不存在的庭院、車庫、街景、陽台、景觀。
6. 不得讓物件看起來與原照片不一致。
7. 不得儲存房屋照片本身。

建議政策文字：

```text
若使用者有提供房屋照片，圖片生成時僅能以使用者提供的房屋照片作為房屋視覺素材，不得重畫成不同房屋，不得新增不存在的外觀、室內、庭院、車庫、街景或景觀，也不得儲存房屋照片本身。
```

### 若使用者未提供房屋照片

必須遵守：

1. 不得生成具體房屋實景。
2. 不得生成假外觀。
3. 不得生成假室內照。
4. 不得生成假街景。
5. 可以使用抽象背景、質感背景、色塊、幾何圖形或非具體房屋的氛圍設計。

建議政策文字：

```text
若使用者未提供房屋照片，不得生成具體房屋實景、假外觀、假室內照或假街景；僅可使用抽象背景、質感背景、色塊、幾何圖形或非具體房屋的氛圍設計。
```

## 文字與資訊防呆政策

圖片生成時禁止新增：

- 電話
- 手機
- LINE ID
- Email
- 完整地址
- 門牌
- 地坪
- 建坪
- 屋齡
- 樓層
- 學區
- 商圈
- 交通時間
- 交流道距離
- 市場、公園、醫院等未提供機能資訊

除非該資訊已存在於已確認物件資料中。

建議政策文字：

```text
圖片生成時不得新增電話、LINE ID、Email、完整地址、門牌或任何未提供的房產資訊；不得自動補坪數、屋齡、樓層、學區、商圈、交通時間或生活機能。
```

## 素材儲存政策

必須保留以下原則：

1. 使用者上傳圖片資料不得被系統儲存。
2. 不得儲存圖片、PDF、QR Code、人物照、房屋照片、名片圖片等非文字素材。
3. 流程內只允許儲存文字資料、素材狀態、合規結果、風格代碼、finalImagePrompt 與圖片生成保護政策。

建議政策文字：

```text
流程內不得儲存使用者上傳的圖片、PDF、QR Code、人物照、房屋照片或名片圖片等非文字素材；僅可儲存文字資料、素材狀態、合規結果、風格代碼、finalImagePrompt 與圖片生成保護政策。
```

## 圖片生成前檢查清單

進入圖片生成前，必須確認：

```text
finalImagePrompt 已存在
qrcodePolicy 已存在
portraitPolicy 已存在
failsafePolicy 已存在
selectedStyle 已存在
已確認素材狀態
已通過合規檢查
```

缺少任一項，不得進入圖片生成。

## 圖片生成後檢查清單

圖片生成後，completeGeneration 前必須檢查：

```text
沒有新增電話
沒有新增 LINE ID
沒有新增完整地址
沒有生成假 QR Code
沒有重畫、變形或偽造 QR Code
人物未變形
未提供人物照時沒有生成虛構人物
沒有生成不存在的房屋實景
沒有自動補未提供坪數、屋齡、樓層、學區、商圈、交通資訊
```

通過後才可呼叫：

```text
completeGeneration
```

## 本階段完成條件

建立圖片生成保護政策後，必須呼叫：

```text
saveImagePolicies
```

成功後才能進入：

```text
IMAGE_GENERATION_READY
```

## 禁止行為

本階段不得：

- 回頭修改物件資料
- 回頭修改風格
- 回頭修改合規結果
- 直接呼叫 completeGeneration
- 跳過圖片生成前檢查
- 儲存圖片、PDF、QR Code、人物照、房屋照片等非文字素材

## nextInstruction

請根據本階段規則，建立圖片生成保護政策。必須保留 QR Code 不重畫、不偽造，人物不變形，未提供人物照不得生成虛構人物，不生成不存在的房屋實景，不新增電話、LINE ID、完整地址或未提供資訊。完成後必須呼叫 saveImagePolicies。
