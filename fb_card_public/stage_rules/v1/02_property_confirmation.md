# 02_property_confirmation｜物件資料確認規則

## 本階段責任

本階段只負責整理使用者提供的物件資料，並等待使用者明確確認。

使用者未明確確認前，不得呼叫 `savePropertyData`。

## 可整理的物件資料欄位

請只根據使用者已提供的資料整理以下內容：

```json
{
  "title": "",
  "price": "",
  "location": "",
  "address": "",
  "layout": "",
  "parking": "",
  "landArea": "",
  "buildingArea": "",
  "age": "",
  "floor": "",
  "highlights": [],
  "contactData": {}
}
```

欄位說明：

- `title`：案名或物件主標。
- `price`：總價或開價。
- `location`：區域，例如彰化縣員林市。
- `address`：地址。若使用者未提供完整地址，不得自行補完整地址。
- `layout`：格局，例如 6房4廳5衛。
- `parking`：車位資訊。
- `landArea`：地坪。
- `buildingArea`：建坪。
- `age`：屋齡。
- `floor`：樓層。
- `highlights`：使用者明確提供的特色。
- `contactData`：使用者明確提供的聯絡文字資料。

## 整理原則

1. 只能使用使用者已提供的文字資料。
2. 不得根據照片自行判斷坪數、屋齡、樓層、格局、建材、學區、商圈或交通資訊。
3. 不得自行補完整地址。
4. 不得自行補電話。
5. 不得自行補 LINE ID。
6. 不得自行補經紀人姓名、營業員證號或公司資料，除非使用者已提供或系統後續有正式店家資料來源。
7. 若資料缺漏，應標示「未提供」，不得猜測。
8. 若使用者提供的資料互相矛盾，必須列出待確認項目。

## 使用者確認規則

整理完成後，必須請使用者確認。

可接受的明確確認包括：

- 確認
- 沒問題
- 正確
- 可以
- 就這樣
- 資料正確
- 使用者明確表示可進下一步

未明確確認前，不得呼叫：

```text
savePropertyData
```

## 呈現格式建議

請用清楚列表呈現：

```text
請確認以下物件資料：

案名：
總價：
地點：
格局：
車位：
特色：
未提供資訊：

確認後我才會儲存物件資料並進入合規檢查。
```

## 禁止行為

本階段不得：

- 進行合規判定
- 提供風格選項
- 建立 finalImagePrompt
- 建立圖片生成政策
- 生成圖片
- 呼叫 completeGeneration
- 儲存圖片、PDF、QR Code、人物照、房屋照片等非文字素材

## 本階段完成條件

使用者明確確認後，必須呼叫：

```text
savePropertyData
```

接著依流程呼叫：

```text
confirmPropertyData
```

## nextInstruction

請根據本階段規則，整理已提供的物件資料，列出可用於圖卡的文字資訊。不得補未提供資料。整理後必須等待使用者明確確認，使用者確認前不得呼叫 savePropertyData。
