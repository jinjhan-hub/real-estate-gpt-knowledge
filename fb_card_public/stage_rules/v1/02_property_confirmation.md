# 02｜Property Confirmation Rule
# ruleKey: 02_property_confirmation
# requiredStage: PROPERTY_CONFIRMATION
# nextAction: savePropertyData
# mustCallNext: waitForUserConfirmationBeforeSavePropertyData

本規則用於「物件資料確認階段」。

本階段目標：
1. 整理使用者提供的物件資料。
2. 整理名片或使用者輸入的營業員聯絡資料。
3. 確認店家 / 公司 / 經紀人揭露資料來源。
4. 在使用者確認前，不得進入合規檢查。
5. 不得自行生成任何未提供的證號、地址、電話或揭露文字。

---

## 一、流程硬鎖

進入本階段時，必須遵守：

1. 不得呼叫 saveComplianceCheck。
2. 不得呼叫 selectImageStyle。
3. 不得呼叫 saveFinalImagePrompt。
4. 不得呼叫 saveImagePolicies。
5. 不得呼叫 generateImage。
6. 不得呼叫 completeGeneration。
7. 不得跳過使用者確認。
8. 不得使用其他 session 的資料。
9. 不得使用測試案例殘留資料。
10. 不得自行補齊任何未提供或無法辨識的資料。

本階段完成條件：

1. 已整理物件資料。
2. 已整理營業員與公司聯絡資料。
3. 已標記缺少或無法辨識的欄位。
4. 已請使用者確認資料。
5. 使用者明確確認後，才可呼叫 savePropertyData。

---

## 二、物件資料整理規則

可整理的物件資料包含：

1. 物件名稱 / 標題
2. 總價
3. 地址或區域
4. 土地坪數
5. 建物坪數
6. 格局
7. 屋齡
8. 樓層
9. 車位
10. 臨路
11. 朝向
12. 物件特色
13. 生活機能
14. 交通條件
15. 學區
16. 其他使用者明確提供的銷售重點

資料來源可以是：

1. 使用者文字輸入
2. 使用者提供的截圖
3. 使用者提供的 DM
4. 使用者提供的 PDF
5. 使用者提供的物件照片中可清楚辨識的文字
6. 使用者明確補充的資料

禁止事項：

1. 不得自行推測價格。
2. 不得自行推測地址。
3. 不得自行推測坪數。
4. 不得自行推測格局。
5. 不得自行推測屋齡。
6. 不得自行補上不存在的車位、學區、建材或生活機能。
7. 不得把範例資料帶入正式測試。
8. 不得使用其他案件資料補齊本案欄位。

若資料缺漏，必須標記為「未提供」或「待補充」。

---

## 三、名片資料讀取規則

若使用者有上傳名片，且名片文字可清楚辨識，允許從名片讀取以下資料：

1. 營業員姓名
2. 營業員電話
3. 營業員證號
4. 公司地址
5. 公司電話
6. 店名或加盟店名稱
7. 公司名稱

讀取原則：

1. 只能讀取名片上清楚可辨識的文字。
2. 不得猜測模糊、殘缺、被遮擋或無法辨識的字號。
3. 營業員證號必須逐字照抄，不得改寫格式。
4. 電話號碼必須逐字照抄，不得自行補碼。
5. 公司地址與公司電話若名片可辨識，可以帶入聯絡資訊。
6. 若名片未提供某項資料，或無法清楚辨識，必須標記為「未提供」，並請使用者補充。
7. 不得使用其他測試案例、範例資料或記憶中的資料補齊。
8. 不得將看起來像證號但無法確認的文字當成正式營業員證號。
9. 不得將名片以外來源的資料誤判為名片資料。

若使用者沒有上傳名片，以下資料必須由使用者自行填入：

1. 營業員姓名
2. 營業員電話
3. 營業員證號
4. 公司地址
5. 公司電話

若使用者未提供上述資料，不得自行生成，必須列為待補充。

## 四、店家資料與名片資料的關係

storeId / code 用於取得店家認證資料與基本揭露資料。

名片用於補充營業員個人聯絡資料與可辨識的公司聯絡資料。

若 API disclosure 與名片資料衝突：

1. 公司名稱、經紀人姓名、經紀人字號、經紀業相關字號，以 API disclosure 為準。
2. 營業員姓名、電話、營業員證號，以清楚可辨識的名片資料或使用者明確輸入為準。
3. 公司地址、公司電話，以清楚可辨識的名片資料或使用者明確輸入為準。
4. 若無法判斷衝突來源，必須停止並請使用者確認。
5. 不得自行選擇看起來比較合理的版本。
6. 不得用記憶、範例或其他店家資料補齊。

---

## 五、揭露資料來源規則

揭露資料分為兩類。

### A. 店家 / 公司 / 經紀人揭露資料

優先來源：

1. storeId / code 綁定資料庫
2. API disclosure 回傳資料

包含：

1. 店家名稱
2. 公司名稱
3. 經紀人姓名
4. 經紀人字號
5. 經紀業相關字號
6. API disclosure 內明確提供的其他揭露資訊

禁止：

1. 不得由 GPT 自行生成。
2. 不得由 GPT 根據店名推測公司名稱。
3. 不得由 GPT 根據縣市推測經紀業字號。
4. 不得使用測試資料。
5. 不得使用其他店家資料。

### B. 營業員 / 個人聯絡資料

允許來源：

1. 名片上清楚可辨識的文字
2. 使用者明確輸入
3. API disclosure 回傳資料

包含：

1. 營業員姓名
2. 營業員電話
3. 營業員證號
4. 公司地址
5. 公司電話

禁止：

1. 不得由 GPT 自行生成營業員姓名。
2. 不得由 GPT 自行生成營業員電話。
3. 不得由 GPT 自行生成營業員證號。
4. 不得由 GPT 自行生成公司地址。
5. 不得由 GPT 自行生成公司電話。
6. 不得補齊名片上模糊不清的數字。
7. 不得使用看似正式但來源不明的證號。
