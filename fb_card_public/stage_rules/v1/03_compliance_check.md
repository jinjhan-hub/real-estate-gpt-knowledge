# 03_compliance_check｜銷售圖卡合規檢查規則

## 本階段責任

本階段只負責檢查已確認物件資料與預計使用文字是否符合銷售圖卡合規要求。

不得處理：
- 風格選擇
- finalImagePrompt
- 圖片生成政策
- 圖片生成
- completeGeneration

## 合規檢查重點

### 一、不得新增未提供聯絡資訊

禁止自動新增：
- 電話號碼
- 手機號碼
- LINE ID
- LINE 官方帳號
- Email
- 社群帳號
- QR Code 內容

除非使用者已明確提供，或已從使用者提供的名片 / 文字素材中整理出來。

### 二、不得新增完整地址

若使用者只提供區域，例如：

```text
彰化縣員林市
```

不得自行補成完整地址。

允許使用：
- 縣市
- 鄉鎮市區
- 使用者明確提供的路段或地標

禁止自行補：
- 門牌號碼
- 詳細巷弄
- 未提供的社區位置
- 未提供的地標距離

### 三、不得補未提供房產資訊

禁止自動補：
- 地坪
- 建坪
- 屋齡
- 樓層
- 朝向
- 面寬
- 臨路寬
- 建材
- 學區
- 商圈
- 市場
- 車站
- 交流道
- 公園
- 醫院
- 捷運、公車、交通時間

除非使用者已明確提供。

### 四、不得誇大或保證

避免使用：
- 保證增值
- 穩賺不賠
- 必漲
- 秒殺
- 絕對稀有
- 最便宜
- 全區最低
- 唯一
- 100% 安全

可改成中性描述：
- 空間條件完整
- 適合重視大坪數與車位的買方評估
- 可作為換屋族參考
- 生活機能需依實際位置與買方需求評估

### 五、揭露資訊檢查

若資料中有下列資訊，應保留於合規可用文字：
- 經紀業名稱
- 經紀人姓名
- 經紀人證號
- 營業員姓名
- 營業員證號
- 店名

若缺少其中部分資訊，不得自行補齊，只能列入 warnings 或待補資料。

### 六、圖片素材合規限制

本階段只做文字合規，不處理圖片本身。

但仍須保留以下限制：
- 不得要求系統儲存使用者上傳圖片。
- 不得要求系統分析 QR Code 內容。
- 不得要求系統從人物照推測身分。
- 不得要求系統從房屋照片推測未提供資料。

## 合規結果格式

請輸出合規檢查結論：

```json
{
  "passed": true,
  "issues": [],
  "warnings": [],
  "allowedText": []
}
```

欄位說明：
- `passed`：是否通過合規檢查。
- `issues`：必須修正的問題。
- `warnings`：提醒事項。
- `allowedText`：確認可使用於圖卡的文字內容。

## 判斷原則

1. 若沒有違規或補資料問題，`passed = true`。
2. 若有新增未提供資訊，`passed = false`。
3. 若只是資料不足但沒有違規，放入 `warnings`。
4. 合規檢查只根據已確認物件資料，不得引入新資訊。
5. 合規未通過，不得進入 style_selection。

## 禁止行為

本階段不得：
- 提供風格清單
- 替使用者選風格
- 建立 finalImagePrompt
- 建立圖片生成保護政策
- 生成圖片
- 呼叫 completeGeneration

## 本階段完成條件

完成合規檢查後，必須呼叫：

```text
saveComplianceCheck
```

## nextInstruction

請根據本階段規則，檢查已確認物件資料與預計使用文字是否符合銷售圖卡合規要求。不得新增電話、LINE ID、完整地址或未提供的房產資訊。完成後必須呼叫 saveComplianceCheck。
