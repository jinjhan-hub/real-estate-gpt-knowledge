# FB 銷售圖卡合規檢查規則｜公版

本檔用於 FB 4:5 房仲銷售圖卡生成前的合規檢查。

適用時機：
使用者完成缺少資料補充後，進入「確認資料」與「選擇風格」之前，必須先讀取本檔。

本檔目的：
避免不動產廣告出現未經確認資料、不實資訊、誇大用語、經紀業揭露不足、經紀人與營業員資訊混用、假 QR Code、假地圖、假街景、假物件照片等問題。

────────────────────
一、刊登前狀態提醒
────────────────────

若內容將作為對外公開廣告使用，必須確認此物件具備合法可刊登狀態。

若使用者未明確確認是否已有委託可刊登：
- 不得主動宣稱「已合法委託」
- 不得主動宣稱「可立即公開刊登」
- 應提醒使用者：對外刊登前，請確認已符合公司與法規所需的委託、審稿或簽章流程。

若使用者明確表示尚未委託、委託不確定、資料未確認：
- 不得生成正式對外廣告圖卡
- 僅可協助整理內部草稿
- 必須標示：此為內部草稿，對外刊登前需完成委託與審稿確認

────────────────────
二、必要揭露資訊檢查
────────────────────

生成 FB 銷售圖卡前，必須確認：

1. 經紀業名稱
2. 經紀人姓名
3. 經紀人證號

若缺少任一項：
- 不得進入風格選擇
- 不得生成圖卡
- 必須請使用者補充

不得：
- 用店名取代經紀業名稱
- 用營業員姓名取代經紀人姓名
- 用營業員證號取代經紀人證號
- 使用預設資料
- 沿用其他案例資料
- 自行補齊任何證號

若屬加盟體系：
- 廣告中應保留加盟店或加盟經營相關標示
- 不得只寫品牌名稱而忽略實際經紀業名稱

────────────────────
三、營業員資訊檢查
────────────────────

若使用者提供或名片中可辨識：

- 營業員姓名
- 營業員證號

必須正確顯示於圖卡資訊區。

營業員證號常見格式：
- (109)登字第362957號
- (111)登字第XXXXXX號
- 登字第XXXXXX號

營業員證號不得誤判為經紀人證號。
經紀人證號不得誤判為營業員證號。
兩者若同時存在，兩者皆需顯示。

────────────────────
四、物件資料真實性檢查
────────────────────

以下資料只能使用使用者明確提供的內容：

- 開價 / 總價
- 地址 / 區域
- 土地坪數
- 建物坪數
- 格局
- 樓層
- 屋齡
- 車位
- 類型
- 學區
- 生活機能
- 交通距離
- 管理費
- 使用分區
- 建照、使照、謄本相關資訊
- 是否全新整理
- 是否可立即入住
- 是否邊間、雙併、獨棟、無共牆
- 是否電梯、孝親房、可停多車

不得從照片自行推測：
- 地址
- 門牌
- 學區
- 交通距離
- 屋況
- 裝潢等級
- 是否全新整理
- 是否採光佳
- 是否通風佳
- 是否安靜
- 是否鄰近市場、公園、學校、車站

若資料來源不明：
- 標示為「未確認」
- 不得放入圖卡
- 不得改寫成肯定句

────────────────────
五、禁止與高風險廣告用語
────────────────────

不得出現以下文字或相近意思：

- 保證增值
- 穩賺不賠
- 投資一定賺
- 必漲
- 秒殺
- 唯一最低
- 全區最低
- 絕對便宜
- 錯過不再
- 內定學區
- 明星學區，除非使用者提供正式依據
- 走路 X 分鐘，除非使用者提供並確認
- 到車站 X 分鐘，除非使用者提供並確認
- 最佳投資標的
- 包租
- 高投報保證
- 未來一定增值
- 政府重大建設保證帶動

可以改成較安全寫法：

- 生活機能便利
- 適合自住規劃
- 空間使用彈性
- 交通動線便利
- 近主要道路
- 適合重視生活機能的買方
- 依使用者提供資料呈現

────────────────────
六、圖像與素材合規檢查
────────────────────

生成圖卡時只能使用：

- 使用者上傳的物件照片
- 使用者上傳的室內照
- 使用者上傳的外觀照
- 使用者上傳的名片
- 使用者上傳的人物照
- 使用者提供並確認的文字資料

不得自行生成：

- 不存在的建物外觀
- 不存在的室內裝潢
- 不存在的街景
- 不存在的地圖
- 不存在的生活圈示意
- 不存在的社區外觀
- 不存在的天空、庭院、車庫、家具作為物件事實
- 假 QR Code
- 假 LINE QR Code
- 假條碼
- 假地標

若缺少物件照片：
- 不得自行補建物或室內圖
- 應要求使用者補照片
- 或僅產生文字排版草稿，不生成正式銷售圖卡

────────────────────
七、合規檢查輸出格式
────────────────────

每次合規檢查後，必須輸出：

【合規檢查結果】

1. 結論：
- 通過 / 不通過 / 需補資料後再檢查

2. 已通過項目：
- 條列說明

3. 需修正項目：
- 條列說明
- 說明原因

4. 需補充資料：
- 條列說明

5. 建議修正文字：
- 主標：
- 副標：
- 賣點 1：
- 賣點 2：
- 賣點 3：

6. 是否可進入風格選擇：
- 可以 / 不可以

若結論不是「通過」：
不得進入風格選擇。
不得生成圖卡。
