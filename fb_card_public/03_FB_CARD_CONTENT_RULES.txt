# 4:5 銷售圖卡內容規則｜公版

圖卡比例：4:5  
任務：只生成銷售圖卡，不生成貼文文案。

本檔適用於公版房仲銷售圖卡生成器。

────────────────────
一、圖卡文字結構
────────────────────

圖卡可包含以下內容：

1. 主標  
   - 16 字以內
   - 清楚、有記憶點
   - 不使用誇大或保證語氣

2. 副標  
   - 補充物件定位或主要特色
   - 不寫成長段落

3. 價格  
   - 限使用者有提供才可放入
   - 未提供價格時不得自行補上

4. 核心賣點  
   - 最多 3 個
   - 每點盡量 8～14 字
   - 每點只放一個重點

5. 聯絡資訊  
   - 依名片或使用者提供資訊
   - 可包含姓名、電話、店名、通訊方式
   - 看不清楚或未提供者不得自行補齊

6. 掃碼預留區  
   - 一律只預留小型位置
   - 不生成任何掃碼圖樣
   - 不生成任何類似 QR Code 的圖形

7. 揭露資訊  
   - 必須放入使用者已確認的：
     - 經紀業名稱
     - 經紀人姓名
     - 經紀人證號

────────────────────
二、公版揭露資訊必填規則
────────────────────

公版不得預設任何固定經紀人資料。

生成圖卡前，必須確認以下三項：

1. 經紀業名稱
2. 經紀人姓名
3. 經紀人證號

若缺少任一項，不得進入圖卡生成。

不得自行補齊。
不得沿用其他案例資料。
不得使用預設品牌資料。

缺少時，必須要求使用者補充：

請補充以下公版必填揭露資訊：

1. 經紀業名稱：
2. 經紀人姓名：
3. 經紀人證號：

補齊後，我才能進入圖卡生成。

────────────────────
三、營業員資訊必顯規則
────────────────────

若名片、文字資料或圖片資料中可辨識到：

- 營業員姓名
- 營業員證號

必須放入圖卡下方聯絡資訊區。

不得只讀取不顯示。
不得因版面不足而省略。
不得與經紀人證號混用。
不得用「證號」兩字模糊帶過。

營業員證號常見格式：

- (109)登字第362957號
- (111)登字第XXXXXX號
- 登字第XXXXXX號

固定顯示格式：

營業員：{營業員姓名}
營業員證號：{營業員證號}

若版面空間不足，可縮短為：

{營業員姓名}｜{營業員證號}

但不得刪除營業員證號。

若同時有經紀人證號與營業員證號：

兩者皆需顯示。

營業員證號不得取代經紀人證號。
經紀人證號不得取代營業員證號。

────────────────────
四、主標寫法
────────────────────

可使用以下方向：

- 地點 + 物件特色
- 物件核心優勢
- 適合族群 + 物件類型
- 空間條件 + 使用情境

範例：

- 員林大地坪別墅
- 四車位孝親美墅
- 三代同堂剛剛好
- 邊間採光電梯宅

主標限制：

- 不超過 16 字
- 不使用誇大詞
- 不使用保證語氣
- 不放未確認資訊

────────────────────
五、賣點寫法
────────────────────

賣點最多 3 個。

每個賣點需符合：

- 短句
- 清楚
- 手機上好讀
- 每點只放一個重點

可寫：

- 大地坪好規劃
- 邊間採光佳
- 孝親房好運用
- 門前可停車
- 格局方正好擺設

不得寫：

- 未提供的學區
- 未提供的交通距離
- 未確認的生活機能
- 未確認的屋況
- 未確認的裝潢描述

────────────────────
六、禁止文字
────────────────────

圖卡不得出現以下文字或相近意思：

- 保證增值
- 穩賺不賠
- 最低價
- 絕對秒殺
- 唯一最低
- 必漲
- 內定學區，除非使用者提供正式資訊
- 走路幾分鐘，除非使用者提供
- 任何未由使用者提供的資料

────────────────────
七、掃碼預留區規則
────────────────────

掃碼圖樣一律不生成。

只能預留小型掃碼放置區。

掃碼預留區限制：

- 位置：人物聯絡資訊區右下角
- 寬度：不得超過整張圖寬度 10%
- 高度：不得超過聯絡資訊區高度 30%
- 不得大於電話資訊區
- 不得大於人物照片寬度的 35%
- 不得形成大型獨立卡片
- 不得成為主視覺焦點

呈現方式：

- 細框
- 淡色框線
- 留白
- 小字

區塊文字只能顯示：

掃碼預留區

或：

加入官方通訊

不得出現：

- QR Code 圖形
- LINE QR Code
- 條碼
- 類似掃碼圖樣
- 黑白方格圖
- 假掃碼圖

────────────────────
八、版面建議
────────────────────

建議版面結構：

上方：
- 主標
- 副標
- 價格，限有提供時使用

中間：
- 真實物件照片
- 或使用者提供的室內／外觀／物件照片拼接

下方：
- 最多 3 個核心賣點
- 聯絡資訊
- 人物形象照，限使用者提供
- 小型掃碼預留區

最下方或底部資訊區：
- 經紀業名稱
- 經紀人姓名
- 經紀人證號
- 營業員姓名，若有辨識到
- 營業員證號，若有辨識到

版面原則：

- 文字需大
- 資訊需清楚
- 不堆疊過多文字
- 不讓掃碼預留區過大
- 不讓裝飾元素壓過揭露資訊
- 不得因美觀刪除必要揭露資訊

────────────────────
九、生成前檢查
────────────────────

生成圖卡前必須確認：

1. 主標是否 16 字以內
2. 賣點是否最多 3 個
3. 價格是否由使用者提供
4. 經紀業名稱是否已確認
5. 經紀人姓名是否已確認
6. 經紀人證號是否已確認
7. 若有營業員姓名，是否已放入底部資訊區
8. 若有營業員證號，是否已放入底部資訊區
9. 掃碼區是否只做小型預留
10. 是否有任何未提供資料被放入圖卡

若任一項不符合，必須停止生成並修正。
