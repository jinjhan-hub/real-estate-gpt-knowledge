# FB 銷售圖卡內容規則｜公版

本 Knowledge 用於「台灣房仲 FB 銷售圖卡生成 GPT｜認證公版」。
用途：規範 FB 4:5 銷售圖卡的版型、內容、人物、照片、QR Code 預留區與揭露資訊。
本檔案僅可在店家認證成功，且流程進入 fb_card_rules 階段後載入。

────────────────────
一、核心任務
────────────────────

本階段只處理 FB 4:5 房仲銷售圖卡的內容規則。

不得處理：
1. FB 長文
2. IG 文案
3. Threads 文案
4. 短影音腳本
5. 周邊機能圖
6. 格局圖
7. 稅務試算
8. 新聞分析
9. 未經使用者確認的延伸內容

所有內容一律使用繁體中文與台灣房仲用語。

────────────────────
二、圖卡基本規格
────────────────────

正式銷售圖卡必須符合：

1. 尺寸比例：FB 4:5 直式圖卡
2. 版面用途：手機瀏覽優先
3. 視覺方向：乾淨、清楚、可信任、資訊易讀
4. 主標必須明確
5. 賣點最多 3 個
6. 不得資訊過滿
7. 不得將價格、坪數、格局、地址等未確認資訊放入圖卡
8. 公版揭露資訊必須放在底部或下方資訊區
9. QR Code 預留區必須出現
10. 若有使用者提供的人物照，正式圖卡應優先安排人物形象區

────────────────────
三、可使用資料來源
────────────────────

圖卡內容只能使用：

1. 使用者本次對話提供且可辨識的物件資料
2. 使用者本次對話上傳的物件照片、截圖、DM、PDF、名片、人物照
3. 使用者本次對話提供且可辨識的電話、LINE ID、營業員資訊
4. verifyStoreAccess 回傳的 disclosure 公版揭露資料
5. loadPublicKnowledge 回傳的規則

不得使用：

1. 模型記憶
2. 過去對話
3. 範例資料
4. 舊店家資料
5. 推測資料
6. 自行補齊的生活機能、交通距離、學區、地圖、街景或室內照

────────────────────
四、禁止自行補資料
────────────────────

未由使用者提供、無法辨識或尚未確認者，不得放入圖卡。

禁止自行補：

1. 價格
2. 坪數
3. 地址
4. 格局
5. 屋齡
6. 樓層
7. 車位
8. 土地分區
9. 學區
10. 交通距離
11. 生活機能
12. 電話
13. LINE ID
14. 公司名稱
15. 經紀業名稱
16. 經紀人姓名
17. 經紀人證號
18. 營業員姓名
19. 營業員證號
20. 室內照
21. 外觀照
22. 街景
23. 地圖
24. 真實 QR Code

若資料不確定，應標示為「不確定資訊」並等待使用者補充或確認，不得自行放入正式圖卡。

────────────────────
五、主標規則
────────────────────

主標必須：

1. 簡短清楚
2. 符合物件實際特色
3. 不誇大
4. 不使用未確認資訊
5. 不使用絕對化、保證型文字
6. 適合放在圖卡上方或主視覺區

可使用方向：

1. 地段型主標
2. 格局型主標
3. 屋況型主標
4. 生活感主標
5. 稀有性主標，但不得誇大
6. 首購、換屋、家庭、投資等需求導向主標，但必須符合資料

不得使用：

1. 保證增值
2. 穩賺不賠
3. 最便宜
4. 唯一
5. 絕對稀有
6. 秒殺
7. 未確認的學區或商圈名稱
8. 未確認的交通時間

────────────────────
六、賣點規則
────────────────────

賣點最多 3 個。

每個賣點必須：

1. 來自使用者提供且可確認的資料
2. 簡短
3. 適合圖卡閱讀
4. 不過度推銷
5. 不使用未確認距離或時間

賣點可包含：

1. 格局特色
2. 屋況特色
3. 坪數特色
4. 地段特色
5. 採光、通風、動線等可由照片合理辨識的特色
6. 車位、電梯、孝親房、邊間、無共牆等已確認資訊
7. 近道路、商圈、學校、市場等已確認資訊

若資料不足，應以保守文字呈現，例如：

1. 格局方正
2. 空間好規劃
3. 採光明亮
4. 生活機能可再依現場確認
5. 適合自住規劃

不得自行寫：

1. 走路幾分鐘
2. 開車幾分鐘
3. 明星學區
4. 未來增值
5. 投報率
6. 租金收益
7. 保證貸款成數
8. 保證可停幾台車

────────────────────
七、價格規則
────────────────────

只有在使用者提供明確價格時，圖卡才可放價格。

可放價格條件：

1. 使用者文字明確提供
2. 截圖中價格清楚可辨識
3. 使用者已確認價格

不可放價格情況：

1. 價格模糊
2. 截圖無法辨識
3. 多個價格來源互相矛盾
4. 使用者尚未確認
5. 只有推測行情

若價格不可放，圖卡不得出現任何價格數字。
可改用：

1. 歡迎洽詢
2. 詳情請洽
3. 價格依使用者確認後放入

不得自行產生價格。

────────────────────
八、照片使用規則
────────────────────

只能使用本次對話中使用者實際上傳的圖片。

可使用：

1. 物件室內照
2. 物件外觀照
3. 591 截圖
4. 樂屋網截圖
5. 房仲 DM
6. 名片
7. 人物照
8. 使用者提供的其他可辨識素材

不得生成或補充：

1. 未提供的室內照
2. 未提供的外觀照
3. 未提供的社區照
4. 未提供的街景
5. 未提供的地圖
6. 未提供的生活圈照片
7. 未提供的裝潢示意圖
8. 未提供的房屋示意圖
9. 未提供的 QR Code

若照片不足，只能用以下方式補版面：

1. 色塊
2. 資訊卡
3. 線條
4. icon
5. 留白
6. 文字排版
7. 幾何區塊
8. 淺色底圖
9. 裝飾性圖形

不得用 AI 自行生成新房屋、新室內、新街景或新地圖。

────────────────────
九、人物使用規則
────────────────────

若使用者未提供人物照，不得生成人物。

若使用者有提供以下任一素材，正式圖卡應優先放入人物形象區：

1. 人物照
2. 名片人物照
3. 可辨識人物素材

人物形象規則：

1. 人物應依照使用者提供照片生成相近專業形象
2. 保留人物臉部特徵、五官比例、髮型氣質與整體辨識感
3. 可做自然商務形象修飾
4. 可安排在圖卡左側、右側、下方或資訊區旁
5. 人物不得壓過物件主資訊
6. 人物不得遮擋主標、賣點、價格、聯絡資訊、公版揭露資訊或 QR Code 預留區

禁止：

1. 憑空生成陌生人物
2. 生成與提供照片明顯不同的新人物
3. 改成卡通人物
4. 改成插畫人物
5. 改成 3D 人物
6. 因為人物可能不夠百分百保真就直接省略人物區
7. 讓人物臉部嚴重變形
8. 讓手指明顯變形
9. 讓人物看起來不像台灣房仲專業形象
10. 讓人物穿著過度浮誇或與房仲形象不符

若人物照解析度較低，仍應以保守方式安排人物形象區，例如半身、側身、遠一點的比例或名片人物小區塊，不得直接省略。

────────────────────
十、QR Code 預留區規則
────────────────────

每張正式 FB 銷售圖卡都必須保留 QR Code 放置區。

注意：
這是「預留空位」，不是生成 QR Code。

QR Code 預留區必須：

1. 固定放在下方聯絡資訊區右側或右下角
2. 使用小型白底或淺色底方框
3. 方框內只能放文字：QR Code 預留區
4. 不得成為主視覺
5. 不得遮擋聯絡資訊
6. 不得遮擋公版揭露資訊
7. 不得省略

禁止：

1. 生成任何 QR Code 圖樣
2. 生成 LINE QR Code
3. 生成條碼
4. 生成黑白方格
5. 生成類似掃碼圖樣
6. 使用假 QR Code
7. 使用不可掃描但看似 QR Code 的裝飾圖樣

建議位置：

1. 下方聯絡資訊區右側
2. 右下角
3. 底部資訊區內的獨立小方框

建議文字：

QR Code 預留區

不得加入其他文字。

────────────────────
十一、公版揭露資訊規則
────────────────────

公版揭露資訊必須來自 verifyStoreAccess 回傳的 disclosure。

欄位對應：

1. disclosure.brokerageName → 經紀業名稱
2. disclosure.brokerName → 經紀人姓名
3. disclosure.brokerLicenseNo → 經紀人證號

圖卡底部或下方資訊區必須放入：

1. 經紀業名稱
2. 經紀人姓名
3. 經紀人證號

禁止：

1. 使用者手動修改公版揭露資訊
2. 用名片資料覆蓋 disclosure
3. 用模型記憶補資料
4. 用舊店家資料補資料
5. 用範例資料補資料
6. 自行修正公司名稱、姓名或證號

若 disclosure 缺少任一項，停止生成，不得製作圖卡。

回覆：

本店家公版揭露資料尚未設定完整，請聯絡管理者更新 Vercel 的 STORE_ACCESS_CONFIG 後重新認證。

────────────────────
十二、營業員與聯絡資訊規則
────────────────────

營業員資訊與聯絡資訊只能使用使用者提供且可辨識的資料。

可使用：

1. 名片上的營業員姓名
2. 名片上的營業員證號
3. 使用者提供的電話
4. 使用者提供的 LINE ID
5. 使用者提供的門市名稱
6. 使用者提供的其他聯絡資訊

不得自行補：

1. 電話
2. LINE ID
3. 營業員姓名
4. 營業員證號
5. 分店地址
6. 公司電話
7. 門市名稱
8. 社群帳號

若名片資訊模糊或無法辨識，必須列為不確定資訊並等待使用者確認。

營業員資訊不得覆蓋 disclosure 公版揭露資訊。
營業員資訊可以與 disclosure 公版揭露資訊同時出現，但必須清楚區分。

────────────────────
十三、版面建議
────────────────────

圖卡建議區塊：

1. 上方：主標
2. 中央：物件照片或主視覺
3. 側邊或照片旁：最多 3 個賣點
4. 下方：價格、聯絡資訊、營業員資訊
5. 底部：公版揭露資訊
6. 右下角或下方聯絡資訊區右側：QR Code 預留區
7. 若有人物照：安排人物形象區，不得遮擋重要資訊

常見版型：

1. 大圖主視覺型
2. 人物加物件型
3. 三賣點資訊卡型
4. 乾淨白底商務型
5. 精品簡報型
6. 科技資訊型
7. 溫暖家庭型

若照片少，建議使用：

1. 單張大圖
2. 一張主圖加資訊卡
3. 人物形象加物件重點
4. 色塊與 icon 輔助
5. 留白設計

若照片多，建議使用：

1. 一張主圖加 2 至 3 張小圖
2. 拼貼式照片牆
3. 主圖搭配重點資訊
4. 不超過 4 張照片，避免過度擁擠

────────────────────
十四、文字密度規則
────────────────────

圖卡文字必須精簡。

建議：

1. 主標 1 句
2. 副標最多 1 句
3. 賣點最多 3 個
4. 每個賣點盡量 8 至 14 字內
5. 價格區清楚
6. 聯絡資訊清楚
7. 揭露資訊可小字但必須可讀
8. QR Code 預留區文字清楚

不得：

1. 塞入完整 FB 文案
2. 塞入過多說明文字
3. 把所有物件資料全部放上去
4. 讓主標、賣點、價格互相搶版面
5. 讓公版揭露資訊小到不可辨識

────────────────────
十五、合規底線
────────────────────

圖卡不得包含：

1. 不實資訊
2. 未確認資料
3. 誇大用語
4. 保證獲利
5. 保證增值
6. 保證貸款
7. 保證收益
8. 絕對化比較
9. 暗示違法用途
10. 隱匿必要揭露資訊
11. 錯誤經紀業名稱
12. 錯誤經紀人姓名
13. 錯誤經紀人證號
14. 假 QR Code
15. 未提供的房屋、室內、街景或地圖

合規檢查未通過，不得生成圖片。

────────────────────
十六、最終輸出前檢查
────────────────────

生成正式圖卡前，必須確認：

1. 店家認證成功
2. disclosure 完整
3. 物件資料已判讀
4. 名片與聯絡資訊已判讀，若有提供
5. 合規檢查通過
6. 使用者已完成最終確認
7. 使用者已選定風格
8. 本 Knowledge 已成功載入
9. image_prompt 已成功載入
10. generation_failsafe 已成功載入
11. 使用者明確說「確認、生成、開始製作、確認生成」
12. QR Code 預留區已納入圖卡規劃
13. 若有人物照，人物形象區已納入圖卡規劃
14. 未使用任何未提供照片
15. 未自行補任何未確認資訊

若任一項不成立，不得生成圖片。
