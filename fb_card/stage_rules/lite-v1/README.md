# 房仲銷售圖卡 Lite v1 Stage Rules

本資料夾提供房仲銷售圖卡 Lite 版的分階段規則，供 GPTs 透過 API 依照目前流程階段取用。

## 適用範圍

- 台灣房仲 FB 4:5 銷售圖卡
- 使用者本輪提供的物件資料、照片、名片、QR Code 或補充文字
- 產圖前的資料整理、合規檢查、風格選擇與提示詞摘要
- 產圖後的人工檢查清單

## 檔案順序

1. `material_check.md`：檢查本輪素材狀態。
2. `property_extract.md`：整理圖卡需要的物件資料，並請使用者確認。
3. `compliance_check.md`：檢查房仲廣告揭露、營業員資訊與素材限制。
4. `style_selection.md`：提供固定 7 種圖卡風格。
5. `image_prompt.md`：組合產圖提示詞與素材保護規則。
6. `final_generation_brief.md`：產圖前的單一權威摘要。
7. `manual_review.md`：產圖後人工檢查清單。

## 共通原則

- 每一階段只處理該階段任務，不預先執行後續階段。
- 只使用使用者本輪提供、明確確認或可靠來源回傳的資料。
- 不得使用其他案件、範例、測試資料或模型記憶補齊房仲揭露資訊。
- 不得要求產圖模型生成假物件照片、假人物、假 QR Code 或不存在的揭露文字。
- 不得要求 API 保存圖片、素材檔或產圖結果物件。

## 對使用者的回覆基準

- 缺資料時，列出缺少項目並用台灣房仲常用語請使用者補充。
- 資料可判讀但有不確定時，標示「待確認」，不得自行補齊。
- 進入產圖前，必須讓使用者確認物件資料、揭露資訊、風格與素材限制。
