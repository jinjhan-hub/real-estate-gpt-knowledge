# image_prompt｜產圖提示詞組合規則

## 階段任務

將已確認的物件資料、合規結果、風格選擇與素材限制組合成產圖提示詞。本階段只產生提示詞內容，不要求 API 保存圖片、素材檔或產圖結果物件。

## 提示詞核心結構

產圖提示詞應包含：

1. 圖卡尺寸與用途：FB 4:5 台灣房仲銷售圖卡。
2. 已選風格代碼與風格描述。
3. 已確認物件資料與主文案。
4. 已確認聯絡與揭露資料。
5. 物件照片使用限制。
6. 人物照片使用限制。
7. QR Code 預留區限制。
8. 文字可讀性與版面要求。

## 物件照片規則

提示詞必須明確寫入：

```txt
這是房仲銷售圖卡排版任務，不是房屋照片生成任務。
房屋、室內、外觀、庭院、車庫、家具、裝潢與景觀畫面，只能使用使用者本輪提供的原始物件照片。
可裁切、拼貼、加框、加陰影、加遮罩、調整亮度與對比，但不得重新生成、重繪、幻想或替換物件照片內容。
不得生成假物件照片。
```

## 人物照片規則

若本輪有提供人物照或名片上有人物照，提示詞必須寫入：

```txt
人物只能使用使用者本輪提供的人物照或名片上的人物照片。
可裁切、去背、加框或放入聯絡資訊區，但不得改變人物身份、五官、髮型、臉型、年齡感或性別。
人物不要變形，不要變成不同的人。
不得生成假人物、假房仲、職業照、半身照或頭像。
```

若未提供人物照，提示詞必須寫入：

```txt
未提供人物照，請不要生成真人房仲、頭像、半身照或任何人物。
```

## QR Code 規則

提示詞必須明確寫入：

```txt
QR Code 區只能預留乾淨空白框，可標示「QR Code 預留區」。
不得生成 QR Code。
不得重繪 QR Code。
不得仿製 QR Code。
不得生成任何類 QR Code 黑白方塊圖樣。
```

## 文字規則

- 圖卡文字必須使用已確認資料。
- 揭露資訊要清楚可讀，不可過小到無法辨識。
- 不得出現未確認電話、證號、地址、公司名稱或經紀人資料。
- 不得出現測試資料、範例資料或其他案件資料。
- 不得使用保證性、絕對性或誤導性廣告文字。

```txt
- 經紀人：

```
