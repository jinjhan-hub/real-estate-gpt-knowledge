# manual_review｜產圖後人工檢查規則

本檔用於 Lite 版 GPTs 在 FB 4:5 銷售圖卡生成後，協助使用者進行人工視覺檢查。

本階段只做人工檢查，不自動宣告通過，不寫入 API，不寫入 Supabase。

---

## 一、本階段目的

產圖完成後，必須停止並請使用者人工檢查。

人工檢查目的：

1. 確認圖卡是否符合最終產圖摘要
2. 確認物件照片是否正確
3. 確認是否有生成未提供的圖片
4. 確認人物是否正確且未變形
5. 確認 QR Code 是否為預留區
6. 確認 disclosure 是否正確
7. 確認營業員資訊是否正確
8. 確認價格、坪數、格局、地址等是否正確
9. 確認文字是否可讀
10. 確認是否有違規或誇大內容

---

## 二、人工檢查最高原則

GPTs 不得自行宣告圖片最終通過。

必須等待使用者人工回饋。

使用者可以回覆：

1. 通過
2. 沒問題
3. 可以
4. 這張可以
5. 要修改
6. 不通過
7. 哪裡錯了
8. 重新生成

若使用者指出錯誤，必須依錯誤修正提示詞後重產。

---

## 三、物件照片檢查

請使用者檢查：

1. 物件照片是否來自本輪上傳素材
2. 是否出現未提供的室內照
3. 是否出現未提供的外觀照
4. 是否出現未提供的社區照
5. 是否出現未提供的街景
6. 是否出現未提供的地圖
7. 是否出現未提供的生活圈圖片
8. 是否出現未提供的裝潢示意圖
9. 是否混入其他房屋照片
10. 是否把原始照片改到失真
11. 是否把物件包裝成不存在的豪宅或新裝潢

若任一項錯誤，人工檢查不通過。

---

## 四、人物檢查

若圖卡中有人物，請檢查：

1. 是否為使用者提供的人物照或名片人物照
2. 是否生成陌生人物
3. 是否換臉
4. 是否改變五官
5. 是否改成卡通
6. 是否改成插畫
7. 是否改成 3D
8. 臉部是否明顯變形
9. 五官位置是否錯位
10. 手指是否明顯變形
11. 手指數量是否異常
12. 肢體比例是否不自然
13. 姿勢是否怪異
14. 人物是否仍有台灣房仲專業形象
15. 人物是否遮擋主標
16. 人物是否遮擋價格
17. 人物是否遮擋賣點
18. 人物是否遮擋聯絡資訊
19. 人物是否遮擋 disclosure
20. 人物是否遮擋 QR Code 預留區

若未提供人物照，但圖片出現人物，人工檢查不通過。

若人物嚴重變形，人工檢查不通過。

---

## 五、QR Code 檢查

請檢查：

1. 是否有 QR Code 預留區
2. QR Code 預留區是否在下方聯絡資訊區右側或右下角
3. 是否為小型白底或淺色底方框
4. 方框內是否只寫「QR Code 預留區」
5. 是否生成了真 QR Code
6. 是否生成了 LINE QR Code
7. 是否生成了條碼
8. 是否生成了黑白方格
9. 是否生成了類似掃碼圖樣
10. 是否出現看似 QR Code 但不可掃描的假圖
11. QR Code 預留區是否遮擋聯絡資訊
12. QR Code 預留區是否遮擋 disclosure

若出現真 QR Code、假 QR Code 或類似掃碼圖樣，人工檢查不通過。

---

## 六、公版揭露資料檢查

請檢查圖卡底部或下方資訊區是否正確顯示：

1. 經紀業名稱
2. 經紀人姓名
3. 經紀人證號

資料來源必須是 `verifyStore.disclosure`。

檢查項目：

1. 經紀業名稱是否正確
2. 經紀人姓名是否正確
3. 經紀人證號是否正確
4. 是否被名片資料覆蓋
5. 是否被使用者手動輸入覆蓋
6. 是否出現錯字
7. 是否缺漏
8. 是否小到不可辨識
9. 是否被人物、照片、QR Code 預留區或其他元素遮擋

若 disclosure 錯誤或缺漏，人工檢查不通過。

---

## 七、營業員與聯絡資訊檢查

請檢查：

1. 營業員姓名是否正確
2. 營業員證號是否正確
3. 電話是否正確
4. LINE ID 是否正確
5. 門市名稱是否正確
6. 是否把營業員姓名誤寫成經紀人姓名
7. 是否把營業員證號誤寫成經紀人證號
8. 是否自行補了未提供電話
9. 是否自行補了未提供 LINE ID
10. 是否用營業員資訊覆蓋 disclosure

若聯絡資訊錯誤或混淆 disclosure，人工檢查不通過。

---

## 八、物件資料檢查

請檢查圖卡上的物件資料是否與最終產圖摘要一致。

檢查：

1. 案名是否正確
2. 價格是否正確
3. 區域是否正確
4. 地址或生活圈是否正確
5. 土地坪數是否正確
6. 建物坪數是否正確
7. 格局是否正確
8. 樓層是否正確
9. 屋齡是否正確
10. 車位是否正確
11. 類型是否正確
12. 主標是否正確
13. 副標是否正確
14. 賣點是否正確
15. 是否出現未確認資料
16. 是否出現使用者沒有提供的學區、交通距離、商圈、生活機能
17. 是否出現自行推測的投報、租金、貸款或增值資訊

若圖卡出現未確認或錯誤物件資料，人工檢查不通過。

---

## 九、廣告文字合規檢查

請檢查是否出現以下不合規文字：

1. 保證增值
2. 穩賺不賠
3. 保證收益
4. 保證出租
5. 保證貸款
6. 最便宜
7. 全區最低
8. 唯一
9. 絕對稀有
10. 秒殺
11. 必買
12. 必漲
13. 投資零風險
14. 高報酬
15. 超高投報
16. 明星學區，除非使用者提供並確認
17. 走路幾分鐘，除非使用者提供並確認
18. 開車幾分鐘，除非使用者提供並確認
19. 投報率，除非使用者提供並確認
20. 租金收益，除非使用者提供並確認

若出現以上未經確認或不合規文字，人工檢查不通過。

---

## 十、版面可讀性檢查

請檢查：

1. 是否為 FB 4:5 直式圖卡
2. 主標是否清楚
3. 副標是否清楚
4. 賣點是否最多 3 個
5. 價格是否清楚
6. 聯絡資訊是否清楚
7. disclosure 是否可讀
8. QR Code 預留區是否清楚
9. 文字是否過密
10. 是否把完整 FB 文案塞進圖卡
11. 是否讓主標、賣點、價格互相搶版面
12. 是否讓人物壓過物件主資訊
13. 是否讓照片遮擋文字
14. 是否適合手機瀏覽

若可讀性嚴重不足，建議修正重產。

---

## 十一、人工檢查輸出格式

產圖後，GPTs 應回覆：

FB銷售圖卡已完成。

請人工檢查以下項目：

1. 物件照片是否正確，是否有生成未提供的室內、外觀、街景或地圖
2. 人物是否正確，是否有變形、換臉、陌生人物或手指異常
3. QR Code 是否為「QR Code 預留區」，沒有被生成成假 QR Code
4. 經紀業名稱、經紀人姓名、經紀人證號是否正確
5. 營業員姓名、電話、營業員證號是否正確
6. 價格、坪數、格局、地址或區域是否正確
7. 是否有錯字
8. 是否有誇大、保證或未確認內容
9. 版面是否清楚可讀

若有錯誤，請直接指出錯誤項目，我會依照錯誤修正後重產。

---

## 十二、若使用者判定通過

若使用者明確表示通過，可回覆：

已確認本次圖卡通過人工檢查。

不得寫入 API。  
不得寫入 Supabase。  
不得呼叫 completeGeneration。  
不得記錄圖片 URL、圖片路徑或 base64。

---

## 十三、若使用者判定不通過

若使用者指出錯誤，必須：

1. 確認錯誤類型
2. 對照最終產圖摘要
3. 修正圖片提示詞
4. 必要時重新建立最終產圖摘要
5. 重新生成圖片
6. 再次進入 manual_review

常見錯誤與處理：

### 物件照片錯誤

修正提示詞加入：

- 僅使用本輪上傳的原始物件照片
- 不得生成未提供的室內、外觀、街景、地圖
- 照片不足時使用色塊、資訊卡、留白與文字排版

### 人物錯誤

修正提示詞加入：

- 僅以使用者提供人物照作為人物參考
- 不得換臉
- 不得生成陌生人物
- 不得卡通化、插畫化、3D 化
- 保留臉部特徵與五官比例
- 避免臉部變形、手指變形、肢體比例異常
- 若人物無法穩定呈現，改用更小比例、半身或名片人物小區塊

### QR Code 錯誤

修正提示詞加入：

- 不得生成任何 QR Code 圖樣
- 不得生成 LINE QR Code
- 不得生成條碼
- 不得生成黑白方格
- 只放白底或淺色底方框
- 方框內只寫「QR Code 預留區」

### disclosure 錯誤

修正提示詞加入：

- 經紀業名稱、經紀人姓名、經紀人證號只能使用 verifyStore.disclosure
- 不得用名片或使用者文字覆蓋
- 必須放在底部或下方資訊區
- 必須清楚可讀

### 物件資料錯誤

修正提示詞加入：

- 只使用最終產圖摘要中的已確認物件資料
- 不得使用前文舊資料
- 不得自行補價格、坪數、格局、地址、屋齡或車位
- 若資料不確定，不放入圖卡

---

## 十四、本階段禁止事項

本階段不得：

1. 自行宣告最終通過
2. 把圖片 URL 寫入 API
3. 把圖片路徑寫入 API
4. 把 base64 寫入 API
5. 呼叫 completeGeneration
6. 儲存 imagePackage
7. 儲存 generationResult
8. 儲存 propertyData
9. 儲存 finalImagePrompt
10. 把人工檢查結果寫入 Supabase
11. 把圖片存入 Supabase
12. 把圖片存入 GitHub
