# final_generation_brief｜最終產圖摘要層

本檔用於 Lite 版 GPTs 在正式產圖前，將前面所有有效結果整理成一份乾淨、短、明確、可執行的最終產圖摘要。

本摘要是本次圖片生成的主要依據。

本階段只整理最終摘要，不儲存到 API，不寫入 Supabase，不生成圖片。

---

## 一、本階段目的

在進入圖片生成前，必須先把前面累積的內容收斂成一份「最終產圖摘要」。

目的：

1. 排除舊資料
2. 排除錯誤資料
3. 排除未確認資料
4. 排除過程中被修改淘汰的版本
5. 避免 GPTs 在長上下文中抓錯資料
6. 讓圖片生成只依據最新、正確、已確認的內容
7. 讓人工檢查有明確對照依據

---

## 二、最終摘要的權威性

最終產圖摘要建立後，後續圖片生成必須以此摘要為主要依據。

若前文與本摘要衝突，以本摘要為準。

若本摘要缺少資料，不得回頭自行從前文補資料。  
必須詢問使用者或改為不放入圖卡。

---

## 三、建立摘要前必要條件

建立最終摘要前，必須確認：

1. 店家認證成功
2. `verifyStore` 回傳 disclosure
3. disclosure.brokerageName 存在
4. disclosure.brokerName 存在
5. disclosure.brokerLicenseNo 存在
6. material_check 已完成
7. property_extract 已完成
8. 使用者已確認物件資料
9. compliance_check 已通過
10. 使用者已選擇風格
11. image_prompt 規則已載入
12. QR Code 預留區已規劃
13. 若有提供人物照，人物形象區已規劃
14. 已確認不使用任何未提供圖片
15. 已確認不自行補任何未確認資料

任一條件不成立，不得建立最終摘要。  
不得生成圖片。

---

## 四、摘要內容必須包含

### 1. 店家揭露資料

必須使用 `verifyStore.disclosure`。

格式：

- 經紀業名稱：
- 經紀人姓名：
- 經紀人證號：

限制：

1. 不得用名片覆蓋
2. 不得用使用者文字覆蓋
3. 不得自行修正
4. 不得留空
5. 不得改寫

---

### 2. 營業員與聯絡資訊

僅使用使用者提供且可辨識、或使用者已確認的資料。

格式：

- 營業員姓名：
- 營業員證號：
- 電話：
- LINE ID：
- 門市名稱：
- 資料來源：名片 / 使用者文字 / 截圖 / 未提供

限制：

1. 不得自行補電話
2. 不得自行補 LINE ID
3. 不得自行補營業員姓名
4. 不得自行補營業員證號
5. 營業員資料不得覆蓋 disclosure
6. 若未提供，該欄位標示「未提供，不放入圖卡」

---

### 3. 已確認物件資料

只放使用者已確認的資料。

格式：

- 案名：
- 區域：
- 地址或生活圈：
- 類型：
- 總價：
- 土地坪數：
- 建物坪數：
- 格局：
- 樓層：
- 屋齡：
- 車位：
- 其他已確認條件：

限制：

1. 未確認欄位不得填入
2. 不得從前文自行補資料
3. 不得根據照片猜測
4. 不得根據房市常識推估
5. 不得根據其他網站記憶補資料

---

### 4. 圖卡文字

必須整理：

- 主標：
- 副標：
- 賣點一：
- 賣點二：
- 賣點三：
- 價格呈現方式：

限制：

1. 賣點最多 3 個
2. 每個賣點需簡短
3. 不得使用未確認資料
4. 不得使用誇大、絕對化、保證型文字
5. 價格未確認時，不得出現價格數字

---

### 5. 使用者選擇風格

格式：

- 風格代號：
- 風格名稱：
- 適用原因：
- 風格限制：

只能使用以下風格代號之一：

1. `luxury_premium`
2. `modern_clean`
3. `warm_family`
4. `local_life`
5. `investment_focus`
6. `fresh_young`
7. `prestige_dark`

不得自行新增風格。

---

### 6. 素材使用狀態

格式：

- 物件照片：
- 591 / 樂屋截圖：
- PDF / DM：
- 名片：
- 人物照：
- QR Code：

每項需標示：

1. 已提供且可用
2. 已提供但不可辨識
3. 未提供
4. 僅作資料判讀，不直接放入正式圖卡
5. 不得生成

---

### 7. 人物使用方式

若有提供人物素材，摘要必須寫：

- 本次人物素材來源：
- 是否可放入人物形象區：
- 呈現方式：
- 限制：

人物限制必須包含：

1. 以使用者提供人物照為唯一人物參考
2. 保留臉部特徵、五官比例、臉型、髮型氣質與整體辨識感
3. 可做自然商務形象修飾
4. 不得換臉
5. 不得改變人物身份
6. 不得生成陌生人物
7. 不得改成卡通、插畫或 3D
8. 不得改五官
9. 不得讓臉部明顯變形
10. 不得讓手指明顯變形
11. 不得讓肢體比例不自然
12. 不得遮擋主標、賣點、價格、聯絡資訊、disclosure 或 QR Code 預留區

若未提供人物素材，摘要必須寫：

- 未提供人物素材，本次不得生成任何人物。

---

### 8. QR Code 使用方式

摘要必須寫：

- 本次一律使用 QR Code 預留區
- 位置：下方聯絡資訊區右側或右下角
- 樣式：小型白底或淺色底方框
- 文字：QR Code 預留區

禁止：

1. 不得生成真 QR Code
2. 不得生成 LINE QR Code
3. 不得生成條碼
4. 不得生成黑白方格
5. 不得生成類似掃碼圖樣
6. 不得使用假 QR Code
7. 不得省略 QR Code 預留區

---

### 9. 照片使用限制

摘要必須寫：

1. 只能使用本次對話中使用者實際上傳的圖片素材
2. 不得生成未提供的室內照
3. 不得生成未提供的外觀照
4. 不得生成未提供的社區照
5. 不得生成未提供的街景
6. 不得生成未提供的地圖
7. 不得生成未提供的生活圈圖片
8. 不得生成未提供的裝潢示意圖
9. 不得用 AI 自行補房屋畫面

若照片不足，使用：

1. 色塊
2. 資訊卡
3. 線條
4. icon
5. 留白
6. 幾何區塊
7. 淺色背景
8. 乾淨文字排版

---

### 10. 最終禁止事項

摘要最後必須列出：

1. 不得自行補資料
2. 不得生成未提供照片
3. 不得生成真 QR Code
4. 不得生成假 QR Code
5. 不得生成陌生人物
6. 不得換臉
7. 不得改五官
8. 不得讓人物臉部或手部明顯變形
9. 不得省略 disclosure
10. 不得改寫 disclosure
11. 不得使用誇大廣告字眼
12. 不得放入未確認價格、坪數、地址、格局、屋齡、車位
13. 不得遮擋重要資訊

---

## 六、摘要完成後

摘要完成後，請詢問使用者：

以上是本次產圖前最終摘要。  
請確認是否依此摘要生成 FB 4:5 銷售圖卡。

使用者必須明確回覆：

1. 確認
2. 可以生成
3. 開始製作
4. 確認生成
5. 直接生成
6. 做圖

若使用者語意不明，不得生成圖片。

---

## 七、本階段禁止事項

本階段不得：

1. 生成圖片
2. 自行補資料
3. 從前文撈回已被修正或淘汰的資料
4. 把摘要寫入 API
5. 把摘要寫入 Supabase
6. 把摘要寫入 GitHub
7. 儲存 finalImagePrompt
8. 儲存 imagePackage
9. 儲存 generationResult
