# final_generation_brief｜產圖前最終摘要

## 階段任務

在產圖前，把前面階段已確認的結果壓縮成單一權威產圖摘要。後續產圖只能依照本摘要執行，不得回頭引用未確認內容、草稿、範例或其他案件資料。

## 摘要原則

- 只保留已確認資料。
- 刪除重複、矛盾、待確認或不該進入產圖的內容。
- 明確列出素材限制，避免產圖模型自行補圖、補人、補 QR Code。
- 若仍有阻擋項目，必須停止並請使用者補充，不得產圖。

## 必含內容

最終摘要必須包含：

1. 圖卡用途與尺寸。
2. 已選風格代碼。
3. 物件主標與副標。
4. 物件核心資料。
5. 三到五個銷售亮點。
6. 聯絡與揭露資料。
7. 物件照片使用規則。
8. 人物照片使用規則。
9. QR Code 預留規則。
10. 廣告文字安全限制。

## 權威性規則

- 若本摘要與前面草稿不同，以本摘要為準。
- 不在本摘要中的資料不得出現在圖卡。
- 標示「待補」的資料不得出現在圖卡，除非使用者已補充並重新確認。
- 不得要求 API 保存圖片、素材檔或產圖結果物件。

## 輸出格式

```txt
產圖前最終摘要：

【用途】
FB 4:5 台灣房仲銷售圖卡

【風格】
style_code：
風格描述：

【物件文案】
主標：
副標：
物件資訊列：

【銷售亮點】
1.
2.
3.

【聯絡與揭露】
營業員：
電話：
營業員證號：
經紀業名稱：
經紀人：
經紀人證號：

【照片限制】
只能使用使用者本輪提供的原始物件照片做排版，不得生成假物件照片或不存在的室內外場景。

【人物限制】
[依本輪人物素材狀態填入：使用指定人物照 / 使用名片人物照 / 不得生成人物]

【QR Code 限制】
只預留乾淨空白 QR Code 區，不得生成、重繪或仿製 QR Code。

【廣告安全】
不得出現未確認資料、測試資料、其他案件資料、保證性或絕對性文字。
```

## 停止條件

若最終摘要仍缺少正式圖卡必要資訊，請回覆：

```txt
目前不可產圖，仍需補充：
- [缺少項目]
```
