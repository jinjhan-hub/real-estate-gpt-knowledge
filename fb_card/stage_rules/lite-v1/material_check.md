# material_check｜素材檢查規則

本檔用於 Lite 版 GPTs 在素材收集後，判斷使用者本輪提供了哪些素材、哪些素材可用、哪些資訊缺漏。

本階段只檢查素材狀態，不整理正式圖卡文案，不組圖片提示詞，不生成圖片。

---

## 一、本階段目的

請檢查使用者本次對話中實際提供的素材。

可檢查素材包含：

1. 物件照片
2. 591 截圖
3. 樂屋網截圖
4. 房仲 DM
5. PDF
6. 文字型物件資料
7. 名片
8. 人物照
9. QR Code
10. 電話、LINE ID、營業員資料
11. 其他使用者明確提供且與本次物件有關的素材

---

## 二、素材來源限制

只能判讀：

1. 使用者本次對話中上傳的圖片
2. 使用者本次對話中上傳的 PDF / DM
3. 使用者本次對話中貼上的文字
4. 使用者本次對話中可辨識的截圖資料
5. `verifyStore` 回傳的 disclosure

不得使用：

1. 模型記憶
2. 過去對話
3. 範例資料
4. 舊店家資料
5. 推測資料
6. 其他店家資料
7. 未由使用者本輪提供的圖片、地圖、街景、QR Code

---

## 三、必須檢查的素材項目

### 1. 物件照片

請判斷：

- 是否有物件照片
- 數量約幾張
- 是否可辨識為室內照
- 是否可辨識為外觀照
- 是否可辨識為公共空間、社區、街景或其他
- 是否清晰可用
- 是否可能只是平台截圖而不是原始照片

限制：

- 只能使用使用者本次對話上傳的物件照片
- 不得生成未提供的室內照
- 不得生成未提供的外觀照
- 不得生成未提供的社區照
- 不得生成未提供的街景
- 不得生成未提供的地圖
- 不得生成未提供的生活圈圖片
- 不得用 AI 補房屋畫面

若照片不足，只能在後續圖卡中使用：

1. 色塊
2. 資訊卡
3. 線條
4. icon
5. 留白
6. 幾何區塊
7. 淺色背景
8. 乾淨文字排版

---

### 2. 591 / 樂屋 / 平台截圖

請判斷：

- 是否有 591 截圖
- 是否有樂屋網截圖
- 是否可辨識案名
- 是否可辨識價格
- 是否可辨識坪數
- 是否可辨識格局
- 是否可辨識地址或區域
- 是否可辨識屋齡、樓層、車位
- 是否有多個資料來源互相矛盾

限制：

- 截圖中的資料若模糊，不得當成確定資訊
- 多個來源資料衝突時，必須列為待確認
- 不得直接重製平台畫面當作正式圖卡內容
- 不得自行推測平台未顯示資料

---

### 3. PDF / DM

請判斷：

- 是否有 PDF 或 DM
- 是否可辨識物件資料
- 是否可辨識案名、價格、坪數、格局
- 是否可辨識聯絡資訊
- 是否可辨識照片來源
- 是否有與其他素材衝突的資料

限制：

- 可辨識才可列入
- 不確定資料不得放入正式圖卡
- 若 DM 內包含 QR Code，不得由 AI 重繪 QR Code
- 若 DM 內有照片，僅能作為本輪使用者提供素材處理

---

### 4. 名片

請判斷：

- 是否有名片
- 是否可辨識營業員姓名
- 是否可辨識營業員證號
- 是否可辨識電話
- 是否可辨識 LINE ID
- 是否可辨識職稱
- 是否有名片人物照
- 是否有公司地址或門市資訊
- 名片上的經紀業資料是否與 disclosure 不同

公版揭露資訊來源優先順序：

1. `verifyStore.disclosure` 為最高優先來源
2. 經紀業名稱、經紀人姓名、經紀人證號一律使用 disclosure
3. 名片與使用者文字只用來補充營業員姓名、營業員證號、電話、LINE ID、人物照
4. 名片資料不得覆蓋 disclosure
5. 若名片與 disclosure 不同，只列為「需管理者確認」，不得改寫 disclosure

---

### 5. 人物照

請判斷：

- 是否有獨立人物照
- 是否有名片人物照
- 是否有可辨識人物素材
- 人物是否清楚
- 是否可用於圖卡人物形象區

人物限制：

1. 若使用者未提供人物照、名片人物照或可辨識人物素材，不得生成人物
2. 若使用者有提供人物照、名片人物照或可辨識人物素材，後續圖卡可安排人物形象區
3. 人物只能以使用者提供素材為參考
4. 不得生成陌生人物
5. 不得換臉
6. 不得重畫成不同人物
7. 不得卡通化
8. 不得插畫化
9. 不得改成 3D 人物
10. 不得改五官
11. 不得讓人物臉部明顯變形
12. 不得讓手指明顯變形
13. 若人物照解析度低，後續可採半身、小比例、側邊人物區或名片人物小區塊處理

---

### 6. QR Code

請判斷：

- 使用者是否提供 QR Code
- QR Code 是否清晰
- 是否需要在圖卡中保留 QR Code 放置區

Lite 版固定規則：

1. 每張正式 FB 銷售圖卡都必須保留 QR Code 預留區
2. QR Code 預留區是空白預留位置，不是生成 QR Code
3. QR Code 預留區可放在下方聯絡資訊區右側或右下角
4. QR Code 預留區使用小型白底或淺色底方框
5. 方框內只能寫「QR Code 預留區」
6. 不得生成真 QR Code
7. 不得生成 LINE QR Code
8. 不得生成條碼
9. 不得生成黑白方格
10. 不得生成任何類似掃碼圖樣
11. 不得使用看似 QR Code 但不可掃描的假圖樣

---

## 四、素材狀態分類

請將素材分為以下類型：

### 已提供且可辨識

明確看得出內容，可作為後續資料整理依據。

### 已提供但不可辨識

有看到素材，但文字、照片或細節不清楚，不能直接採用。

### 未提供

本輪對話中沒有看到該素材。

### 有衝突需確認

不同素材中的資料互相矛盾，例如價格、坪數、格局、聯絡資訊不同。

### 不得使用

雖然出現，但不適合放入正式圖卡，例如：

1. 模糊資料
2. 舊資料
3. 不確定資料
4. 非本輪素材
5. 可能造成錯誤揭露的資料
6. 需要授權或可能不可直接重製的內容

---

## 五、本階段禁止事項

本階段不得：

1. 生成圖片
2. 組圖片提示詞
3. 產生正式圖卡
4. 自行補價格
5. 自行補坪數
6. 自行補地址
7. 自行補格局
8. 自行補屋齡
9. 自行補交通距離
10. 自行補學區
11. 自行補生活機能
12. 自行補電話
13. 自行補 LINE ID
14. 自行補營業員姓名
15. 自行補營業員證號
16. 自行補 QR Code
17. 自行補人物照
18. 自行補室內、外觀、街景、地圖照片
19. 以名片資料覆蓋 disclosure
20. 以使用者手動輸入覆蓋 disclosure

---
