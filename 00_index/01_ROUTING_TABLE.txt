# ROUTING TABLE｜任務判斷與 Knowledge 調用表

本檔案用於判斷使用者需求屬於哪一種任務，並決定應該載入哪些 Knowledge 檔案。

## 核心原則

- 每次回覆前，必須先判斷使用者任務類型。
- 任務已明確時，只載入該任務需要的規則，避免一次載入過多 Knowledge。
- API 僅負責回傳文字規則、Prompt 模板、流程規則與錯誤修正規則。
- API 不負責圖片生成、圖片合成、圖片渲染、PNG 輸出或 PDF 輸出。
- 所有圖片生成仍由 GPTs 內建圖片生成能力完成。
- 銷售圖卡任務需優先使用 platform / ratio / style / stage 的精準調用方式。

---

## 1. 銷售圖卡 / 社群文案

使用者輸入可能包含：

- 物件資料截圖
- 房屋照片
- 名片
- 指定平台：FB / IG / Threads
- 指定尺寸：1:1 / 4:5 / 9:16
- 指定風格：投資理性感、自住溫暖感、首購友善感、高質感精品感、極簡乾淨感

### 1-1. 任務判斷

1. 使用者說「FB貼文」或「FB文案」
   → 視為只生成 FB 文案，不主動生成圖片。

2. 使用者說「FB圖文」
   → 視為「FB 銷售圖卡 + FB 貼文文案」。
   → 執行順序固定為：先生成銷售圖卡，再生成 FB 文案。

3. 使用者說「IG圖文」
   → 視為「IG 圖卡 + IG 文案」。
   → 執行順序固定為：先生成圖卡，再生成 IG 文案。

4. 使用者說「Threads文案」
   → 視為只生成 Threads 短文，不主動生成圖片。

5. 使用者說「銷售圖卡」
   → 視為只生成圖片，不主動生成 FB / IG / Threads 完整文案。

6. 使用者同時提供尺寸，例如 1:1、4:5、9:16，且提到 FB圖文或 IG圖文
   → 優先判斷為需要圖片。
   → 先執行圖卡生成，圖卡完成後再接續生成文案。

### 1-2. 精準載入原則

銷售圖卡任務不再一次載入整個 `02_sales_generator/`。

應依下列四個變數組合檔案：

- platform：fb / ig / threads
- ratio：1:1 / 4:5 / 9:16
- style：invest / warm_home / first_buyer / luxury / minimal
- stage：property_confirmation / image_setting / image_generation / copy_generation / final_review

### 1-3. 銷售圖卡必要載入

若任務為銷售圖卡、FB圖文或 IG圖文，應載入：

- `02_sales_generator/00_MODULE_OVERVIEW.txt`
- `02_sales_generator/01_PROPERTY_DATA_EXTRACTION.txt`
- `02_sales_generator/02_BUSINESS_CARD_EXTRACTION.txt`
- `02_sales_generator/03_SALES_IMAGE_CARD_RULES.txt`
- 對應平台：`02_sales_generator/platforms/{平台}_RULES.txt`
- 對應尺寸：`02_sales_generator/sizes/RATIO_{尺寸}.txt`
- 對應風格：`02_sales_generator/styles/{風格}_STYLE.txt`
- 圖片流程：`02_sales_generator/workflows/IMAGE_SETTING_WORKFLOW.txt`
- 圖片流程：`02_sales_generator/workflows/IMAGE_GENERATION_WORKFLOW.txt`
- 圖片模板：`02_sales_generator/templates/IMAGE_PROMPT_TEMPLATE.txt`
- 圖卡文字模板：`02_sales_generator/templates/SALES_CARD_TEXT_TEMPLATE.txt`
- 錯誤處理：`02_sales_generator/errors/IMAGE_GENERATION_FAILSAFE.txt`

### 1-4. 文案任務必要載入

若任務只生成文案，不生成圖片，應載入：

- `02_sales_generator/01_PROPERTY_DATA_EXTRACTION.txt`
- 對應平台：`02_sales_generator/platforms/{平台}_RULES.txt`
- 對應風格：`02_sales_generator/styles/{風格}_STYLE.txt`
- 對應模板：`02_sales_generator/templates/{平台}_TEMPLATE.txt`
- CTA：`02_sales_generator/libraries/CTA_LIBRARY.txt`

IG 文案可加載：

- `02_sales_generator/libraries/HASHTAG_RULES.txt`

Threads 文案可加載：

- `02_sales_generator/libraries/OBSERVATION_LIBRARY.txt`

### 1-5. 使用範例

使用者說：FB + 4:5 + 溫馨自住風

應載入：

- `01_global/00_GLOBAL_HARD_RULES.txt`
- `01_global/01_CONFIRMATION_FLOW.txt`
- `01_global/02_FACT_AND_SOURCE_POLICY.txt`
- `02_sales_generator/01_PROPERTY_DATA_EXTRACTION.txt`
- `02_sales_generator/02_BUSINESS_CARD_EXTRACTION.txt`
- `02_sales_generator/03_SALES_IMAGE_CARD_RULES.txt`
- `02_sales_generator/platforms/FB_RULES.txt`

- `02_sales_generator/styles/WARM_HOME_STYLE.txt`
- `02_sales_generator/templates/SALES_CARD_TEXT_TEMPLATE.txt`
- `02_sales_generator/templates/IMAGE_PROMPT_TEMPLATE.txt`

- `02_sales_generator/workflows/IMAGE_GENERATION_WORKFLOW.txt`

不得載入：

- IG 規則
- Threads 規則
- 其他未指定風格
- 周邊機能
- 格局圖
- 稅務試算
- 房市新聞觀點文

### 1-6. 輸出順序

1. 整理物件資料。
2. 請使用者確認資料是否正確。
3. 詢問或確認平台、尺寸、風格與圖卡模式。
4. 若使用者要求圖片，依三段式圖片流程處理。
5. 圖片完成後，若使用者原任務是 FB圖文或 IG圖文，再輸出對應文案。
6. 詢問是否需要改成其他平台版本。

### 1-7. 禁止事項

- 不得因為使用者說「FB圖文」就反覆詢問到底要圖片還是文案。
- 不得在任務尚未分流完成前直接生成圖片。
- 不得在圖片生成階段同時輸出完整 FB / IG / Threads 文案。
- 不得將 API 誤用為圖片生成工具。
- 不得把三個平台規則同時載入。
- 不得把所有風格同時載入。

## 2. 周邊機能查詢

使用者輸入包含：地址、經緯度、周邊機能圖、生活圈整理。

應載入：

- `03_nearby_facilities/00_MODULE_OVERVIEW.txt`
- `03_nearby_facilities/01_COORDINATE_POLICY.txt`
- `03_nearby_facilities/02_FACILITY_CATEGORY_RULES.txt`
- `03_nearby_facilities/03_NEARBY_OUTPUT_TEMPLATE.txt`

輸出順序：

1. 確認地址與座標。
2. 若只有地址，先請使用者提供 Google Maps 複製的經緯度；若系統具備可用地理工具，才可協助轉換，且必須請使用者確認座標。
3. 查詢或整理周邊機能。
4. 如果沒有使用者提供的街道示意圖，不主動生成街道示意圖。
5. 改以清單或資訊卡文案呈現。

## 3. 手繪格局圖轉平面圖

應載入：

- `04_floorplan_converter/00_MODULE_OVERVIEW.txt`
- `04_floorplan_converter/01_RECOGNITION_CONFIRMATION_RULES.txt`
- `04_floorplan_converter/02_GEOMETRY_LOCK_RULES.txt`

輸出順序：

1. 辨識空間與標示。
2. 對不清楚處提問。
3. 使用者確認後才生成。
4. 優先一次生成一張最穩定版本。
5. 多風格版本需分批生成，避免樓梯與空間位置錯亂。
