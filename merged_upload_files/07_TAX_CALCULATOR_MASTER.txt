# TAX CALCULATOR OVERVIEW｜稅務試算器總覽

本模組保存使用者稅務試算器專案的規格。

本模組只處理：

- 稅務試算器功能規格
- HTML / CSS / JavaScript 頁面結構
- GitHub Pages 部署規則
- 列印與 PDF 版面規則
- 使用者介面與欄位流程

本模組不提供正式稅務建議，不保證實際稅額。

---

## 專案

1. 房地合一稅試算器：`index.html`
2. 土地增值稅試算器：`land-tax-calculator.html`
3. 入口頁面：太平洋房屋員林站前店稅務試算入口

## 頁面檔名規則

- 房地合一稅試算器：`index.html`
- 土地增值稅試算器：`land-tax-calculator.html`
- 若新增入口頁，需確認是否會取代既有 `index.html`
- 檔名建議使用英文、數字與連字號
- 避免使用中文檔名、空格或特殊符號，降低 GitHub Pages 路徑錯誤

## 共同需求

- Mobile-first
- GitHub Pages 可部署
- 單檔 HTML 優先，方便使用者上傳與維護
- 可列印 A4
- 可另存 PDF
- 列印時浮水印在最上層
- 不使用深色列印背景
- 輸入欄位易懂
- 按鈕距離足夠，避免手機誤觸
- 結果區清楚
- 結果需標示「試算」性質
- 加入免責提醒

## 品牌文字

預設品牌文字：

- 太平洋房屋員林站前店
- 巫建穎製作

實際品牌、店名、公司名稱或姓名若使用者更新，應以最新提供內容為準。

不得使用記憶中的品牌資訊自行補齊。

## 稅務資料來源原則

- 稅率、免稅額、扣除額、CPI、法規條件等，需以主管機關、稅捐機關或使用者提供資料為準。
- 若資料可能已更新，需提醒使用者確認最新公告。
- 不得將試算器結果宣稱為正式稅額。
- 不得保證免稅、節稅或適用特定稅率。
- 若使用者提供財政部或地方稅務局試算結果作比對，應優先檢查輸入欄位、CPI、公式、日期與適用條件差異。

## 開發與修改規則

修改既有試算器時：

- 不得任意刪除既有功能。
- 不得任意更改檔名。
- 不得破壞 GitHub Pages 可部署性。
- 不得移除列印、PDF、浮水印、免責提醒等既有需求。
- 修改公式、稅率或欄位時，需清楚說明修改點。
- 若使用者指出與官方試算結果不同，應先檢查公式與輸入資料，不直接假設官方錯誤。

## 列印與 PDF 規則

- A4 版面需清楚。
- 列印背景以白底為主。
- 不使用深色背景。
- 浮水印需在最上層，但不得遮蔽重要數字。
- 結果表格不可超出頁面。
- 列印按鈕建議放在試算結果下方。
- 手機畫面與列印畫面可使用不同 CSS。

## 免責提醒

試算器需顯示以下概念：

「本工具僅供初步試算與教學參考，實際稅額、適用稅率、免稅條件與申報結果，仍需依主管機關、稅捐機關或專業人士認定為準。」

# HOUSE-LAND TRANSACTION TAX RULES｜房地合一稅試算器規格

## 頁面名稱

太平洋房屋站前店房地合一稅試算器

## 模組定位

本模組用於保存「房地合一稅試算器」的功能規格與介面需求。

本工具僅供初步試算與教學參考，不保證實際稅額。  
實際適用稅率、課稅所得、免稅額、成本費用認定與申報結果，仍需依主管機關、稅捐機關或專業人士認定為準。

---

## 頁面檔名

- 預設檔名：`index.html`
- 若此頁作為 GitHub Pages 首頁，需確認不會覆蓋其他入口頁。
- 若另設入口頁，需重新確認房地合一稅試算器連結路徑。

---

## 主要輸入

### 日期資料

- 購入日期
- 出售日期

### 金額資料

- 購入價格
- 出售價格
- 成本與費用
- 可扣除項目
- 土地漲價總數額或其他依法可扣除項目，若使用者需要加入

### 條件資料

- 是否為境內居住者
- 是否符合自住相關條件
- 是否可能適用自住優惠
- 是否為 105/01/01 前取得
- 是否有合法成本費用憑證

以上條件不得由系統自行判斷為「一定適用」，只能依使用者輸入與試算規則提示。

---

## 日期格式

支援民國年格式：

- `108/03/15`
- `1080315`

輸入：

- `1040321`

應自動轉成：

- `104/03/21`

日期輸入規則：

- 民國年需轉換為西元年進行程式計算。
- 月份需介於 1 至 12。
- 日期需符合該月份有效日期。
- 出售日期不可早於購入日期。
- 日期錯誤時需顯示提醒，不得繼續試算。

---

## 持有期間

- 依購入日期與出售日期自動計算持有期間。
- 結果區需顯示購入日期與出售日期。
- 結果區需顯示持有年數與月數。
- 適用稅率需依持有期間與相關條件判斷。
- 提醒：實務上持有期間與課稅適用，仍需依謄本登記時間、交易資料與稅捐機關認定為準。

---

## 適用稅率規則

- 稅率表應獨立維護，不要散落在程式各處。
- 稅率、持有期間級距、境內外身分適用規則，需標示資料來源與更新日期。
- 若稅率或級距未確認，不得宣稱為最新正式版本。
- 若使用者提供主管機關資料，應以使用者提供並確認的版本為準。

結果區應顯示：

- 判斷依據
- 持有期間
- 身分條件
- 適用稅率
- 注意事項

---

## 成本與費用

功能需求：

- 支援多筆項目。
- 每筆包含項目名稱與金額。
- 可新增項目。
- 可刪除項目。
- 可自動加總。
- 金額需允許整數輸入。
- 空白或非數字需提示錯誤。
- 提醒需有發票、收據或合法憑證。

成本與費用項目可包含：

- 取得成本
- 改良費用
- 移轉相關費用
- 仲介費
- 代書費
- 其他使用者自行輸入項目

不得自行幫使用者判定某項費用一定可扣除。

---

## 3% 推計費用提醒

若使用者未輸入費用，或輸入費用低於系統推計提醒門檻，可顯示提示：

「若未提示因取得、改良及移轉而支付的費用者，得按成交價額 3% 計算，並以 30 萬元為限。若實際支付費用較高且能提供合法支出憑證，可核實認定。」

注意：

- 此規則需標示資料來源與更新日期。
- 若法規更新，需同步修正。
- 試算器可提示使用者比較「實際輸入費用」與「推計費用」，但不得自動判定哪一種一定適用。
- 若使用者選擇推計費用，結果區需顯示計算方式。

---

## 105/01/01 前取得提醒

若購入時間早於民國 105 年 1 月 1 日，可顯示：

「購入時間早於 105 年 1 月 1 日，可能不適用房地合一稅新制，仍需依實際案件、取得原因、移轉情形與主管機關認定。」

注意：

- 不得直接判定一定免課房地合一稅。
- 不得直接將稅額歸零。
- 應提示使用者進一步確認適用制度。

---

## 自住優惠提醒

自住優惠不可簡化為一律 0 稅率。

需依以下條件與金額邏輯判斷：

- 是否符合自住條件
- 是否符合持有與設籍等相關限制
- 是否符合交易前後相關規定
- 課稅所得是否超過免稅額
- 超過免稅額部分是否仍需計稅

注意：

- 不可只因使用者勾選「自住」就直接把稅額歸零。
- 不可只因持有滿 6 年就直接套用 0%。
- 結果區需清楚顯示是否「可能適用」以及仍需確認的條件。
- 若條件未完整輸入，應顯示「自住優惠條件尚未完整確認」。

---

## 計算結果區

結果區需顯示：

- 購入日期
- 出售日期
- 持有期間
- 身分條件
- 適用稅率
- 出售價格
- 購入價格
- 成本與費用明細
- 推計費用提示，若有
- 課稅所得
- 預估稅額
- 自住優惠提示，若有
- 105/01/01 前取得提示，若有
- 注意事項
- 免責提醒

---

## 結果文字提醒

結果區需顯示：

「本結果僅供初步試算，不代表正式稅額。實際課稅所得、適用稅率、免稅額、成本費用認定與申報結果，仍需依稅捐機關核定為準。」

---

---

## 開發注意事項

- 稅率表、免稅額、推計費用規則應集中管理，方便後續更新。

- 不得直接假設官方試算工具錯誤。

## 頁面檔名

`land-tax-calculator.html`

## 模組定位

本模組用於保存「土地增值稅試算器」的功能規格與介面需求。

---

## 主要輸入欄位

### 土地基本資料

- 土地類型：都市 / 非都市
- 土地宗地面積（平方公尺）
- 公告現值（元／每平方公尺）
- 前次移轉現值（元／每平方公尺）
- 前次移轉年月
- 移轉或設典範圍：分子 / 分母
- 消費者物價指數 dropdown

### 欄位輸入限制

- 面積需為正數。
- 公告現值需為正數。
- 前次移轉現值需為正數。
- 移轉或設典範圍分母不得為 0。
- 移轉或設典範圍不得大於全部持分，除非使用者明確說明。
- 前次移轉年月需為有效年月。
- CPI 未選擇時，不得直接試算。

---

## 版面順序

使用者偏好的欄位順序：

1. 土地類型
2. 土地宗地面積
3. 公告現值
4. 移轉或設典範圍
5. 前次移轉年月
6. 前次移轉現值
7. 消費者物價指數

---

## CPI 規則

- 需使用稅務專用物價指數概念。
- CPI 應以稅捐機關或主管機關適用資料為準。
- 若串接政府 API，需確認資料欄位是否與稅捐機關試算工具一致。
- 若 CPI dropdown 使用內建資料，需標示資料來源與更新日期。
- 若使用者指出與財政部或地方稅務局工具不同，需優先檢查：
  1. CPI 是否一致
  2. 面積是否一致
  3. 移轉或設典範圍是否一致
  4. 公告現值是否使用每平方公尺金額
  5. 前次移轉現值是否使用每平方公尺金額
  6. 土地類型是否正確
  7. 公式與級距是否一致
  8. 是否有自用住宅、重購退稅或其他減免條件未納入

# PRINT AND UI RULES｜列印與介面規格

本檔定義稅務試算器的手機介面、列印版、PDF 輸出與 GitHub Pages 部署規則。

適用頁面：

- `index.html`：房地合一稅試算器
- `land-tax-calculator.html`：土地增值稅試算器
- 入口頁面，若使用者另行建立

---

## Mobile-first

介面需以手機使用為優先。

### 欄位設計

- 手機上欄位要大。
- 輸入框高度需足夠，避免難以點選。
- 字體不可太小。
- 數字欄位應使用適合手機輸入的格式。
- 日期欄位需有清楚格式提示。
- 金額欄位需有單位提示，例如「元」。
- 坪數、平方公尺、每平方公尺等單位需標示清楚。

### 按鈕設計

- 按鈕距離要足夠，避免誤觸。
- 「帶入範例」「清除」「開始試算」三個按鈕不可太靠近。
- 主要按鈕需明顯。
- 危險操作，例如清除資料，需避免使用者誤點。
- 手機上按鈕需適合拇指操作。

### 閱讀性

- 表單區塊需分段。
- 結果區要清楚。
- 錯誤提醒要靠近相關欄位。
- 不要讓使用者滑太久才看到重要結果。
- 手機版不使用過寬表格。

---

## 結果區

- 結果要在同頁清楚呈現。
- 結果區需出現在使用者點擊「開始試算」後。
- 結果區需包含輸入摘要、計算結果、注意事項與免責提醒。
- 列印按鈕放在試算結果最下方。
- 結果可直接列印或另存 PDF。
- 若欄位缺漏或格式錯誤，不得產出正式試算結果。
- 若試算涉及未確認條件，結果區需標示「待確認」或「僅供初步參考」。

---

## A4 列印

### 基本規則

- 使用白底。
- 不要深色背景。
- 字體大小足夠。
- 表格不可超出頁面。
- 列印版需適合 A4。
- 頁面邊界需保留足夠空間。
- 不要讓重要結果被分頁切斷。
- 列印版不顯示不必要的操作按鈕。

### 列印內容

列印版需包含：

- 頁面名稱
- 品牌文字
- 輸入資料摘要
- 試算結果
- 成本或費用明細，若有
- 注意事項
- 免責提醒
- 製作來源或浮水印

### 浮水印規則

- 浮水印需在最上層。
- 浮水印不得遮住重要數字。
- 浮水印不得影響結果可讀性。
- 浮水印文字需依使用者最新提供品牌資料為準。
- 若浮水印與表格重疊，應降低透明度或調整位置，但仍需可見。

---

## PDF 輸出

- 使用瀏覽器列印功能另存 PDF。
- PDF 版面需與 A4 列印一致。
- 不得出現深色背景。
- 不得遺漏免責提醒。
- 不得讓結果表格被截斷。
- 若使用者要求下載 PDF，優先確保列印 CSS 正確。

---

## CSS 規則

需分開設計：

### 螢幕版 CSS

- 適合手機瀏覽。
- 表單清楚。
- 按鈕大、間距足夠。
- 結果區易讀。

### 列印版 CSS

使用 `@media print` 控制：

- 隱藏操作按鈕。
- 使用白底。
- 調整字體大小。
- 避免表格超出 A4。
- 控制分頁。
- 顯示浮水印。
- 保留免責提醒。

---

## GitHub Pages

- 靜態 HTML / CSS / JS 優先。
- 優先使用單檔 HTML，方便部署與維護。
- 不依賴後端服務，除非使用者明確要求 API 串接。
- 檔名避免中文與空格。
- 檔名避免特殊符號。
- 修改檔名時，需同步更新入口頁連結。

入口頁可連結：

- `index.html`
- `land-tax-calculator.html`

若使用者已部署 GitHub Pages，不得任意更改檔名或路徑，避免原本連結失效。

---

## 修改既有程式時的限制

- 不得任意刪除原本功能。
- 不得移除列印功能。
- 不得移除 PDF 需求。
- 不得移除浮水印。
- 不得移除免責提醒。
- 不得破壞手機版排版。
- 不得破壞 GitHub Pages 可部署性。
- 修改 UI 時需保留原本試算邏輯，除非使用者明確要求調整公式。
- 修改公式時需說明修改原因與影響範圍。

- 網頁結果區下方
- A4 列印版下方
- PDF 輸出版本下方

建議文字：

「本工具僅供初步試算與教學參考，實際稅額、適用稅率、免稅條件與申報結果，仍需依主管機關、稅捐機關或專業人士認定為準。」
